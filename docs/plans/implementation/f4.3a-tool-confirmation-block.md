# F4.3a Tool Confirmation Block Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` for this slice. Implementer and reviewer subagents must all use model `gpt-5.5`. Implementer subagents must use `superpowers:test-driven-development`; reviewers run in the order spec / plan compliance first, then code quality / testing / regression. `superpowers:executing-plans` is only a fallback when subagents cannot be dispatched or the task boundary cannot be passed precisely.

**Goal:** Implement the Narrative Feed tool-confirmation block so the current active run can allow or deny one specific high-risk tool action inline, show denied follow-up runtime semantics from the new projection contract, keep paused or historical confirmations non-submittable, and preserve the Inspector as a detail surface rather than the primary action surface.

**Architecture:** Extract the tool-confirmation block into a focused feed component plus a narrow action helper. `FeedEntryRenderer` remains the dispatch boundary, but the block itself owns action wording, follow-up rendering, API submission, query invalidation, error display, and Inspector entry affordance. Reuse backend truth from the top-level `tool_confirmation` projection, TanStack Query invalidation for workspace/session refresh, and the existing SSE reducer semantics so no separate frontend state machine or speculative local mutation is introduced.

**Tech Stack:** React 18, TypeScript strict mode, TanStack Query, existing workspace projections and SSE reducer, Vitest, Testing Library, existing frontend CSS baseline.

```tsx
import { apiRequest, type ApiRequestOptions } from "../../api/client";
import type { ToolConfirmationFeedEntry } from "../../api/types";

export type ToolConfirmationDecision = "allow" | "deny";

export type ToolConfirmationDecisionResponse = {
  tool_confirmation: ToolConfirmationFeedEntry;
};

export function submitToolConfirmationDecision(
  entry: ToolConfirmationFeedEntry,
  decision: ToolConfirmationDecision,
  options?: ApiRequestOptions,
): Promise<ToolConfirmationDecisionResponse> {
  const action = decision === "allow" ? "allow" : "deny";
  return apiRequest(`/api/tool-confirmations/${entry.tool_confirmation_id}/${action}`, {
    ...options,
    method: "POST",
  });
}
```
