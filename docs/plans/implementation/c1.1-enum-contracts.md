# C1.1 Enum Contracts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task in the main agent session. Steps use checkbox (`- [ ]`) syntax for tracking. Do not use subagent handoff, Git write actions, PR creation, merge, tag, or branch cleanup for this repository slice.

**Goal:** Implement the first global enum contract surface for Function One V1 so later schemas, projections, events, and frontend mocks share stable machine values.

**Architecture:** Add a focused domain enum module backed by Python `StrEnum`, then expose the same enum classes through `backend/app/schemas/common.py` for Pydantic schema modules. The test suite locks exact values where the split-plan says "only contains" and locks required supersets where the split-plan or backend spec says "at least supports".

**Tech Stack:** Python 3.11 `StrEnum`, Pydantic v2 serialization behavior, pytest through the repo-local `uv run` environment.
