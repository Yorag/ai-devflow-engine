**Goal:** Implement soft-delete session history commands so a Session without an active run can be removed from product-visible history, without deleting runtime facts, audit facts, or repository state.

**Architecture:** Extend `SessionService` with a focused delete command that checks active-run status against the existing run lifecycle truth, flips the existing control-plane visibility fields, and records stable audit outcomes for success, blocked deletion, missing session, and repeated deletion. Expose the command through `DELETE /api/sessions/{sessionId}` and return the already-defined `SessionDeleteResult` schema with OpenAPI coverage.

**Tech Stack:** FastAPI, SQLAlchemy 2.x, Pydantic v2, pytest, repository-local `uv run`.

Service behavior:
- `list_visible_sessions(project_id=..., trace_context=...)` returns visible sessions only, ordered the same way as existing `list_project_sessions()`.
- `assert_session_deletable(session_id=..., trace_context=...)` loads the Session row without applying the visible-only filter so repeated delete can be distinguished from missing Session.
- If the Session row does not exist, raise `SessionServiceError(ErrorCode.NOT_FOUND, SESSION_NOT_FOUND_MESSAGE, 404)` and record rejected audit `session.delete.rejected`.
- If the Session exists but `is_visible` is already `False`, raise `SessionServiceError(ErrorCode.VALIDATION_ERROR, SESSION_ALREADY_REMOVED_MESSAGE, 409)` and record rejected audit `session.delete.rejected`.
- If `current_run_id` is `None`, the Session is deletable.
- If `current_run_id` is not `None`, load the `PipelineRunModel` from the runtime session. If the run exists and `RunStateMachine.is_active_run_status(run.status)` is `True`, return a blocked `SessionDeleteResult` instead of mutating the Session.
- If a first-run startup publication is still pending for the Session, treat that pending publication as an active-run blocker even when `current_run_id` is still `None`, and return the same blocked `SessionDeleteResult` with the pending `run_id` as `blocking_run_id`.
- If `current_run_id` points to a missing runtime row, treat that as not active for this slice rather than inventing a second blocking rule.
- Successful deletion sets `is_visible = False`, sets `visibility_removed_at = now`, updates `updated_at = now`, commits the control session, and returns:

```python
SessionDeleteResult(
    session_id=model.session_id,
    project_id=model.project_id,
    visibility_removed=True,
    blocked_by_active_run=False,
    blocking_run_id=None,
    error_code=None,
    message="Session removed from regular product history.",
)
```

- Active-run blocked deletion does not mutate the Session and returns:

```python
SessionDeleteResult(
    session_id=model.session_id,
    project_id=model.project_id,
    visibility_removed=False,
    blocked_by_active_run=True,
    blocking_run_id=run.run_id,
    error_code="session_active_run_blocks_delete",
    message="Session has an active run.",
)
```

- The command must not clear `current_run_id`, must not change Session status, and must not touch runtime/event/log domain records.
- The guarded delete write path must also fail closed if a pending startup publication appears after the delete eligibility check but before the visibility update.
- `PublicationBoundaryService.begin_startup_publication()` must acquire the pending startup claim against current `SessionModel` visibility/draft/current-run state at insertion time so delete and first-run startup cannot interleave into a hidden Session plus pending startup publication.

API behavior:
- Add `DELETE /api/sessions/{sessionId}` in `backend/app/api/routes/sessions.py`.
- Response model: `SessionDeleteResult`.
- Success response code: `200`.
- Error responses: `404`, `409`, `422`, `500` with `ErrorResponse`.
- A blocked active-run deletion returns HTTP `200` with `blocked_by_active_run=True`, because the split-plan requires stable command semantics and a structured blocked result, not a transport-level error.
- Missing Session returns `404`.
- Repeated deletion returns `409`.

        self.records: list[dict[str, Any]] = []

    def record_command_result(self, **kwargs: Any) -> object:
        self.records.append({"method": "record_command_result", **kwargs})
        return object()

        self.records.append(

                **kwargs,

    def delete_session(
        self,
        *,
        session_id: str,
        trace_context: TraceContext,
    ) -> SessionDeleteResult:
        model, run = self.assert_session_deletable(
            session_id=session_id,
            trace_context=trace_context,
        )
        if run is not None and RunStateMachine.is_active_run_status(run.status):
            self._audit_service.record_command_result(
                actor_type=AuditActorType.USER,
                actor_id="api-user",
                action="session.delete",
                target_type="session",
                target_id=model.session_id,
                result=AuditResult.BLOCKED,
                reason=SESSION_DELETE_BLOCKED_MESSAGE,
                metadata={
                    "session_id": model.session_id,
                    "project_id": model.project_id,
                    "status": model.status.value,
                    "current_run_id": model.current_run_id,
                    "blocking_run_id": run.run_id,
                    "visibility_removed": False,
                },
                trace_context=trace_context,
            )
            return SessionDeleteResult(
                session_id=model.session_id,
                project_id=model.project_id,
                visibility_removed=False,
                blocked_by_active_run=True,
                blocking_run_id=run.run_id,
                error_code=SESSION_DELETE_BLOCKED_ERROR_CODE,
                message=SESSION_DELETE_BLOCKED_MESSAGE,
            )

        timestamp = self._now()
        model.is_visible = False
        model.visibility_removed_at = timestamp
        model.updated_at = timestamp
        self._session.add(model)
        self._session.flush()
        try:
            self._audit_service.record_command_result(
                actor_type=AuditActorType.USER,
                actor_id="api-user",
                action="session.delete",
                target_type="session",
                target_id=model.session_id,
                result=AuditResult.SUCCEEDED,
                reason=None,
                metadata={
                    "session_id": model.session_id,
                    "project_id": model.project_id,
                    "status": model.status.value,
                    "current_run_id": model.current_run_id,
                    "visibility_removed": True,
                    "visibility_removed_at": timestamp.isoformat(),
                },
                trace_context=trace_context,
            )
            self._session.commit()
        except Exception:
            self._session.rollback()
            raise
        return SessionDeleteResult(
            session_id=model.session_id,
            project_id=model.project_id,
            visibility_removed=True,
            blocked_by_active_run=False,
            blocking_run_id=None,
            error_code=None,
            message=SESSION_DELETE_SUCCESS_MESSAGE,
        )
