# H4.5 Pause/Resume Checkpoint Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Use `superpowers:test-driven-development` for every production or test change. Fallback to `superpowers:executing-plans` only if subagents cannot be scheduled or the task boundary cannot be supplied precisely. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement `POST /api/runs/{runId}/pause` and `POST /api/runs/{runId}/resume` so the current active run can pause from `running`, `waiting_clarification`, `waiting_approval`, and `waiting_tool_confirmation`, persist a resumable checkpoint record, and resume the same run and same graph thread with correct paused-projection behavior.

**Architecture:** Extend `RunLifecycleService` into the H4.5 command owner for pause/resume by combining runtime-boundary calls, recovery-checkpoint persistence, session/run status updates, event emission, and audit/log integration. Keep pause/resume out of `control_item` semantics: use `DomainEventType.RUN_PAUSED` / `RUN_RESUMED` for run/session updates, and refresh existing `approval_request` / `tool_confirmation` feed entries with non-actionable or actionable payloads when the paused state changes those UI semantics.

**Tech Stack:** Python 3.11+, FastAPI, SQLAlchemy, Pydantic v2, existing multi-SQLite `DatabaseManager`, `RuntimeOrchestrationService`, `AuditService`, `JsonlLogWriter`, `EventStore`, `pytest` through `uv run`.

## Confirmed Conflict Resolution

The split-plan row mentions `ControlRecordService.append_control_item()`, but the current backend spec and event matrix do not authorize new pause/resume `control_item` semantics:

- `control_item.control_type` remains limited to `clarification_wait`, `rollback`, and `retry`.
- `RunPaused` and `RunResumed` already project through `session_status_changed`, not `control_item_created`.
- The user explicitly approved the narrower interpretation for H4.5 in this session.

Therefore this implementation plan **does not** introduce new pause/resume `RunControlRecordType` values or top-level `control_item` entries. Pause/resume state changes are carried by:

- `DomainEventType.RUN_PAUSED`
- `DomainEventType.RUN_RESUMED`
- refreshed `approval_request` events when a pending approval becomes paused/non-actionable or actionable again
- refreshed `tool_confirmation` events when a pending tool confirmation becomes paused/non-actionable or actionable again
- a persisted `StageArtifactModel(artifact_type="recovery_checkpoint")` process record holding the resumable checkpoint metadata

## Behavior Decisions

- `pause` is valid only for the current active run whose `SessionModel.current_run_id` matches the `runId`, and whose run status is one of:
  - `running`
  - `waiting_clarification`
  - `waiting_approval`
  - `waiting_tool_confirmation`
- `pause` changes `PipelineRun.status` and `SessionModel.status` to `paused`.
- `pause` does **not** introduce a new `StageStatus.PAUSED`; the current `StageRun.status` remains the pre-pause waiting/running value so resume can restore the exact waiting contract without inventing a new stage enum.
- `resume` is valid only for the current active run when `PipelineRun.status` and `SessionModel.status` are `paused`.
- The pre-pause run state is restored from the latest persisted `recovery_checkpoint` artifact:
  - paused-from-`running` resumes to `running`
  - paused-from-`waiting_clarification` resumes to `waiting_clarification`
  - paused-from-`waiting_approval` resumes to `waiting_approval`
  - paused-from-`waiting_tool_confirmation` resumes to `waiting_tool_confirmation`
- `RuntimeOrchestrationService.pause_thread()` receives `workspace_snapshot_ref=run.workspace_ref` as the current stable workspace reference available in this branch baseline.
- `save_resume_checkpoint()` persists a `StageArtifactModel` with:
  - `artifact_type = "recovery_checkpoint"`
  - `payload_ref = checkpoint.checkpoint_id`
  - `process["run_status_before_pause"]`
  - `process["session_status_before_pause"]`
  - `process["stage_status_before_pause"]`
  - `process["checkpoint"]` serialized from `CheckpointRef`
  - `process["graph_thread_ref"]`
  - `process["workspace_snapshot_ref"]`
- `pause` appends `DomainEventType.RUN_PAUSED` with a `session_status_changed` payload.
- `resume` appends `DomainEventType.RUN_RESUMED` with a `session_status_changed` payload.
- When the paused run was waiting for approval:
  - the pending `ApprovalRequestModel` row remains `pending`
  - `pause` appends a refreshed `APPROVAL_REQUESTED` event with `is_actionable = false` and paused disabled reason
  - `resume` appends a refreshed `APPROVAL_REQUESTED` event with `is_actionable = true` and `disabled_reason = null`
- When the paused run was waiting for tool confirmation:
  - the pending `ToolConfirmationRequestModel` row remains `pending`
  - `pause` appends a refreshed `TOOL_CONFIRMATION_REQUESTED` event with `is_actionable = false` and paused disabled reason
  - `resume` appends a refreshed `TOOL_CONFIRMATION_REQUESTED` event with `is_actionable = true` and `disabled_reason = null`
- No new approval result, tool confirmation result, control item, or system status semantics are introduced by H4.5.

def pause_run(
    self,
    *,
    run_id: str,
    actor_id: str,
    trace_context: TraceContext,
) -> RunCommandResult:
    session, run, stage = self._load_active_run_context(run_id)
    self._assert_can_pause(session=session, run=run)
    timestamp = self._now()
    pause_trace = trace_context.child_span(
        span_id=f"runtime-pause-{run.run_id}",
        created_at=timestamp,
        session_id=session.session_id,
        run_id=run.run_id,
        stage_run_id=stage.stage_run_id,
        graph_thread_id=run.graph_thread_ref,
    )
    self._audit_service.require_audit_record(
        actor_type=AuditActorType.USER,
        actor_id=actor_id,
        action="runtime.pause",
        target_type="run",
        target_id=run.run_id,
        result=AuditResult.ACCEPTED,
        reason="Pause accepted.",
        metadata={"run_id": run.run_id, "status_before": run.status.value},
        trace_context=pause_trace,
        rollback=self._rollback_all,
        created_at=timestamp,
    )
    checkpoint_result = self._runtime_orchestration.pause_thread(
        thread=self._build_thread_ref(run=run, stage=stage),
        stage_run_id=stage.stage_run_id,
        stage_type=stage.stage_type,
        workspace_snapshot_ref=run.workspace_ref,

def resume_run(
    self,
    *,
    run_id: str,
    actor_id: str,
    trace_context: TraceContext,
) -> RunCommandResult:
    session, run, stage = self._load_active_run_context(run_id)
    self._assert_can_resume(session=session, run=run)
    checkpoint_artifact = self._latest_recovery_checkpoint(run_id=run_id)
    checkpoint = CheckpointRef.model_validate(checkpoint_artifact.process["checkpoint"])
    pre_pause_status = RunStatus(checkpoint_artifact.process["run_status_before_pause"])
    timestamp = self._now()
    resume_trace = trace_context.child_span(
        span_id=f"runtime-resume-{run.run_id}",
        created_at=timestamp,
        session_id=session.session_id,
        run_id=run.run_id,
        stage_run_id=stage.stage_run_id,
        graph_thread_id=run.graph_thread_ref,
    )
    self._audit_service.require_audit_record(
        actor_type=AuditActorType.USER,
        actor_id=actor_id,
        action="runtime.resume",
        target_type="run",
        target_id=run.run_id,
        result=AuditResult.ACCEPTED,
        reason="Resume accepted.",
        metadata={"run_id": run.run_id, "status_before": run.status.value},
        trace_context=resume_trace,
        rollback=self._rollback_all,
        created_at=timestamp,
    )
    self._runtime_orchestration.resume_thread(
        thread=self._build_thread_ref(run=run, stage=stage, paused=True),
        checkpoint=checkpoint,
        trace_context=resume_trace,
    )
    run.status = pre_pause_status
    run.updated_at = timestamp
    session.status = RunStateMachine.project_session_status(pre_pause_status)
    session.updated_at = timestamp
    self._append_run_status_event(
        DomainEventType.RUN_RESUMED,
        session=session,
        run=run,
        trace_context=resume_trace,
    )
    self._refresh_pending_wait_entry_for_resume(
        pre_pause_status=pre_pause_status,
        run=run,
        trace_context=resume_trace,
    )

    request: Request,
    control_session: Session = Depends(get_control_session),
    runtime_session: Session = Depends(get_runtime_session),
    event_session: Session = Depends(get_event_session),
