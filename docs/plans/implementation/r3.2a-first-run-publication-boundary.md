**Goal:** Add the missing prerequisite infrastructure boundary that lets first-run startup write multi-store product truth without exposing half-started public state before the final publish point.

**Architecture:** Introduce a control-plane publication gate for first-run startup. `RunLifecycleService.start_first_run()` will stage runtime, graph, and event truth under a pending publication record, then publish visibility only after the staged product set is complete. Workspace, timeline, and SSE read paths must consult the publication gate so unpublished startup rows stay invisible even if they already exist in `runtime.db`, `graph.db`, or `event.db`.

## Problem Statement

Current `RunLifecycleService._commit_all()` commits `runtime.db`, `control.db`, `event.db`, and `graph.db` sequentially. That means:

- a late commit failure can leave partial committed startup truth
- a startup retry must reason about leftovers across multiple stores
- the current public read paths can observe rows directly from `runtime.db` and `event.db` without a shared visibility gate

`R3.2a` fixes that prerequisite gap without collapsing the architecture back into a single SQLite file.

## Scope

In scope:

- Add a control-plane publication gate for first-run startup visibility.
- Stage startup runtime/event/graph rows before they become publicly visible.
- Publish startup visibility only after the staged set is complete.
- Keep failed or abandoned staged startup rows invisible to workspace, timeline, and SSE readers.
- Add cleanup / retry semantics for stale pending startup publications.
- Refactor `RunLifecycleService.start_first_run()` to use the new boundary.

## Proposed Data Shape

Add a new control-plane model:

```python
class StartupPublicationModel(ControlBase, TimestampMixin):
    __tablename__ = "startup_publications"

    publication_id: Mapped[str] = mapped_column(String(80), primary_key=True)
    session_id: Mapped[str] = mapped_column(
        String(80),
        ForeignKey("sessions.session_id"),
        nullable=False,
        index=True,
    )
    run_id: Mapped[str] = mapped_column(String(80), nullable=False, index=True)
    stage_run_id: Mapped[str] = mapped_column(String(80), nullable=False)
    publication_state: Mapped[str] = mapped_column(
        String(40),
        nullable=False,
    )  # pending | published | aborted
    pending_session_id: Mapped[str | None] = mapped_column(
        String(80),
        nullable=True,
        unique=True,
        index=True,
    )
    published_at: Mapped[datetime | None] = mapped_column(
        DateTime(timezone=True),
        nullable=True,
    )
    aborted_at: Mapped[datetime | None] = mapped_column(
        DateTime(timezone=True),
        nullable=True,
    )
    abort_reason: Mapped[str | None] = mapped_column(Text, nullable=True)
```

Rationale:

- control-side publication state is the single public gate
- staged product rows remain in their existing responsibility stores
- workspace / timeline / SSE consult the control gate before exposing runtime / event rows

Do not add a second product truth source for `PipelineRun` or events. `StartupPublicationModel` is only a visibility gate, not the business truth.

## Proposed Service Surface

Create `backend/app/services/publication_boundary.py` with:

```python
@dataclass(frozen=True)
class PublishedStartupVisibility:
    publication_id: str
    session_id: str
    run_id: str
    stage_run_id: str


class PublicationBoundaryServiceError(RuntimeError):
    def __init__(self, error_code: ErrorCode, message: str, status_code: int) -> None:
        self.error_code = error_code
        self.message = message
        self.status_code = status_code
        super().__init__(message)


class PublicationBoundaryService:
    def __init__(
        self,
        *,
        control_session: Session,
        runtime_session: Session,
        graph_session: Session,
        event_session: Session,
        audit_service: Any,
        log_writer: Any,
        now: Callable[[], datetime] | None = None,
    ) -> None: ...

    def begin_startup_publication(
        self,
        *,
        session_id: str,
        run_id: str,
        stage_run_id: str,
        trace_context: TraceContext,
    ) -> StartupPublicationModel: ...

    def publish_startup_visibility(
        self,
        *,
        publication_id: str,
        session_id: str,
        run_id: str,
        stage_run_id: str,
        trace_context: TraceContext,
        published_at: datetime,
    ) -> PublishedStartupVisibility: ...

    def abort_startup_publication(
        self,
        *,
        publication_id: str,
        session_id: str,
        run_id: str,
        reason: str,
        trace_context: TraceContext,
        aborted_at: datetime,
    ) -> None: ...

    def visible_run_ids_for_session(
        self,
        *,
        session_id: str,
    ) -> set[str]: ...

    def hidden_run_ids_for_session(
        self,
        *,
        session_id: str,
    ) -> set[str]: ...

    def assert_run_visible(
        self,
        *,
        run_id: str,
    ) -> str: ...
```

Behavior rules:

- `begin_startup_publication()` creates a `pending` publication row before startup staging writes proceed and uses `pending_session_id` as the per-session startup claim guard.
- `publish_startup_visibility()` verifies the startup product set exists in runtime / graph / event stores, then atomically updates `control.db`:
  - `Session.status = running`
  - `Session.current_run_id = run_id`
  - `Session.latest_stage_type = requirement_analysis`
  - `StartupPublicationModel.publication_state = published`
  - `published_at = published_at`
- `abort_startup_publication()` removes staged runtime / graph / event rows for `run_id`, clears any `Session` pointer that still references the staged run, then marks the publication `aborted`.
- `visible_run_ids_for_session()` returns runtime runs except those explicitly hidden by a non-published startup publication row.
- `hidden_run_ids_for_session()` exposes the internal hide set used by workspace, timeline, and SSE readers so legacy runs without startup publication rows stay visible.
- `assert_run_visible()` rejects direct timeline or future inspector access to unpublished runs.

## Read-Path Contract

To make the boundary real, product readers must not expose unpublished startup rows.

Required changes:

- `WorkspaceProjectionService.build_run_summaries()` must filter runtime runs by the hidden-run set for that Session, keeping legacy runs without startup publication rows visible.
- `WorkspaceProjectionService._build_narrative_feed()` must drop events whose `run_id` is hidden.
- `TimelineProjectionService.get_run_timeline()` must reject a run id that is not published.
- `TimelineProjectionService.build_timeline_entries()` must only replay events for runs that are not hidden for the Session.
- SSE stream route must re-check the hidden-run set per replay batch and suppress unpublished startup events.

Do not loosen this by saying “the route should not know the run id yet”. The workspace route already scans all runtime runs by `session_id`, so without these filters the public-semantic guarantee is false.
