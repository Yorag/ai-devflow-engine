## Frontend Design Gate

- Register: product UI. Design serves a work-focused application surface.
- Context load: `impeccable` project context loader reported no `PRODUCT.md` and no `DESIGN.md`. Per slice rules, this does not block F0.1; the project tone is recorded here instead of creating a new design context document.
- Physical scene sentence: a platform developer reviews an AI delivery workflow from a desktop browser during focused implementation work, in normal office lighting, needing quick orientation and stable controls more than brand expression.
- Color strategy: Restrained. Use tinted neutral surfaces with one blue accent for route focus and primary affordance; avoid decorative gradients, heavy card walls, and landing-page composition.
- Tone baseline: quiet, professional, high-information product workspace; dense but readable; predictable navigation; no marketing hero.
- Reference boundary: follow familiar product-shell patterns from Linear/Figma/Notion-style tools in terms of clarity and component discipline; do not copy branding, visual assets, or business semantics.
- F0.1 visual scope: a simple SPA frame with a top tool strip, route navigation, home empty surface, and console placeholder. It establishes typography, colors, focus states, responsive behavior, and testable page landmarks.
- Non-goals: no Project/Session sidebar, no three-column workspace, no Narrative Feed, no Composer, no Inspector, no settings modal, no business data, no API client, no Zustand store, no SSE.
- Responsive strategy: below `720px`, route navigation wraps under the brand row and page padding tightens; text wraps with `overflow-wrap`; no fixed text widths that can force overflow.
- Accessibility strategy: use semantic landmarks (`banner`, `main`, `navigation`), visible focus states, sufficient contrast in restrained colors, descriptive route names, and no hover-only interactions.
- Implementation hardening: include default, hover, focus, active, and responsive states for links/buttons; avoid nested cards; prevent long route/page text from overflowing with `min-width: 0` and `overflow-wrap: anywhere`.

```typescript
import { QueryClientProvider } from "@tanstack/react-query";
import { render, type RenderResult } from "@testing-library/react";
import type { ReactElement } from "react";
import { RouterProvider } from "react-router-dom";

import { createQueryClient } from "./query-client";
import { createTestRouter } from "./router";

type RenderWithAppProvidersOptions = {
  route?: string;
};

export function renderWithAppProviders(
  ui: ReactElement | null,
  options: RenderWithAppProvidersOptions = {},
): RenderResult {
  const queryClient = createQueryClient();

  if (ui) {
    return render(
      <QueryClientProvider client={queryClient}>{ui}</QueryClientProvider>,
    );
  }

  const router = createTestRouter([options.route ?? "/"]);

  return render(
    <QueryClientProvider client={queryClient}>
      <RouterProvider router={router} />
    </QueryClientProvider>,
  );
}
```

import React from "react";
import { createRoot, type Root } from "react-dom/client";

import { App } from "./App";
import "./styles/global.css";

export function mountEngineeringBaseline(rootElement: HTMLElement): void {
  rootElement.dataset.engineeringBaseline = "ready";
}

type RenderApp = (rootElement: HTMLElement) => Root;

export function mountApp(
  rootElement: HTMLElement,
  renderApp: RenderApp = createRoot,
): Root {
  const root = renderApp(rootElement);

  root.render(
    <React.StrictMode>
      <App />
    </React.StrictMode>,
  );

  return root;
}

const rootElement = document.getElementById("root");

if (rootElement) {
  mountApp(rootElement);
}
