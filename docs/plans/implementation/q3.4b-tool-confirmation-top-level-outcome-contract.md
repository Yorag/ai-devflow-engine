**Goal:** Expose stable `deny_followup_action` and `deny_followup_summary` on top-level `tool_confirmation` payloads in workspace query, run timeline query, and SSE replay so clients can render deny outcomes without inferring runtime state.

**Architecture:** Reuse the H4.4b runtime-owned deny-followup source already persisted on `ToolConfirmationRequestModel`, validate the top-level `ToolConfirmationFeedEntry` contract, hydrate AL02-owned read surfaces at query and SSE time, and keep the scope limited to top-level projection contracts plus the narrowly approved producer-path compatibility fix required to emit schema-valid denied events.

**Tech Stack:** Python 3.11, FastAPI, Pydantic v2, SQLAlchemy 2.x, pytest through `uv run python -m pytest`.
