**Goal:** Build the first usable frontend workspace shell with a project/session sidebar, a Narrative Workspace slot, and a default-closed Inspector slot.

**Architecture:** `ConsolePage` becomes a thin composition point that renders `WorkspaceShell`. `WorkspaceShell` owns mock-backed query wiring and layout state, while `ProjectSidebar` and `SessionList` render navigation/status semantics from F2.2 fixtures through query hooks. The slice does not add backend routes, mutate API contracts, or implement settings/templates/run actions beyond visible shell affordances.

**Tech Stack:** React 18, Vite, TanStack Query, Vitest, Testing Library, CSS in `frontend/src/styles/global.css`.

## Frontend Design Gate

**Register:** Product UI.

**Impeccable context:** The repository does not contain `PRODUCT.md` or `DESIGN.md`; `node C:\Users\lkw\.codex\skills\.system\impeccable\scripts\load-context.mjs` returned `hasProduct: false` and `hasDesign: false`. Use the split specs as product context.

**Project tone:** Quiet, professional, high-information-density workspace UI. Users operate the local console during development work on a desktop monitor, with occasional narrow-screen review. Use the existing light, cool-neutral OKLCH palette and restrained accent usage.

**Layout strategy:**
- Desktop: fixed app-height three-column shell, left rail `280px-340px`, center flexible workspace, right Inspector slot `360px-480px` represented by a closed rail button/state.
- Narrow screen: left and right areas collapse into drawer placeholders. F2.3 only needs structural placeholders, not a fully interactive drawer system.
- Avoid marketing hero layout, nested cards, decorative blobs, one-note saturated color, and card walls.

**State and copy strategy:**
- Left rail must show Project Switcher, Load Project entry, current Project summary, New Session entry, Session list, and default delivery mode summary.
- Active/blocked controls are visible but non-destructive. Project removal and Session deletion controls render as disabled/blocking affordances when the current fixture implies default project or active run.
- Narrative Workspace is a slot with empty/current session summary only. It must not implement F2.5 template empty state.
- Inspector is closed by default and must not replace the main Narrative Workspace.

**Post-implementation checks:**
- Keyboard focus visible on shell controls.
- Long project paths and session names wrap or truncate without layout breakage.
- Status labels remain legible on desktop and mobile.
- No visible text describes implementation internals, feature slices, keyboard shortcuts, or testing mechanics.

`ProjectSidebar.tsx`:

```tsx
import type { ApiRequestOptions } from "../../api/client";
import {
  useProjectDeliveryChannelQuery,
  useProjectSessionsQuery,
  useProjectsQuery,
} from "../../api/hooks";
import type { ProjectRead } from "../../api/types";
import { SessionList } from "./SessionList";

type ProjectSidebarProps = {
  request: ApiRequestOptions;
};

export function ProjectSidebar({ request }: ProjectSidebarProps): JSX.Element {
  const projectsQuery = useProjectsQuery({ request });
  const projects = projectsQuery.data ?? [];
  const currentProject = projects[0] ?? null;
  const sessionsQuery = useProjectSessionsQuery(currentProject?.project_id ?? "", {
    request,
  });
  const deliveryQuery = useProjectDeliveryChannelQuery(
    currentProject?.project_id ?? "",
    { request },
  );

  return (
    <aside className="workspace-sidebar" aria-label="Project and session sidebar">
      <ProjectSwitcher projects={projects} currentProject={currentProject} />
      <button className="workspace-button workspace-button--secondary" type="button">
        Load project
      </button>
      {currentProject ? (
        <section className="project-summary" aria-label="Current project summary">
          <div>
            <span>Sessions</span>
            <strong>{sessionsQuery.data?.length ?? 0}</strong>
          </div>
          <div>
            <span>Default delivery</span>
            <strong>{deliveryQuery.data?.delivery_mode ?? "unknown"}</strong>
          </div>
        </section>
      ) : null}
      <button className="workspace-button" type="button">
        New session
      </button>
      <button
        className="workspace-button workspace-button--danger"
        type="button"
        disabled={currentProject?.is_default ?? true}
        aria-label={
          currentProject?.is_default
            ? "Default project cannot be removed"
            : `Remove ${currentProject?.name}`
        }
      >
        Remove project
      </button>
      <SessionList sessions={sessionsQuery.data ?? []} />
    </aside>
  );
}

function ProjectSwitcher({
  projects,
  currentProject,
}: {
  projects: ProjectRead[];
  currentProject: ProjectRead | null;
}): JSX.Element {
  return (
    <section className="project-switcher" aria-label="Project switcher">
      <p className="workspace-eyebrow">Project</p>
      <h2>{currentProject?.name ?? "No project loaded"}</h2>
      <p>{currentProject?.root_path ?? "Load a local project to begin."}</p>
      <label>
        <span>Switch project</span>
        <select value={currentProject?.project_id ?? ""} onChange={() => undefined}>
          {projects.map((project) => (
            <option key={project.project_id} value={project.project_id}>
              {project.name}
            </option>
          ))}
        </select>
      </label>
    </section>
  );
}
```

`SessionList.tsx`:

```tsx
import type { SessionRead, SessionStatus } from "../../api/types";

type SessionListProps = {
  sessions: SessionRead[];
};

export function SessionList({ sessions }: SessionListProps): JSX.Element {
  return (
    <section className="session-list" aria-label="Session list">
      <div className="session-list__header">
        <h2>Sessions</h2>
        <span>{sessions.length}</span>
      </div>
      {sessions.map((session) => (
        <article className="session-list-item" key={session.session_id}>
          <div>
            <h3>{session.display_name}</h3>
            <p>{formatStatus(session.status)}</p>
          </div>
          <div className="session-list-item__actions">
            <button type="button" aria-label={`Rename ${session.display_name}`}>
              Rename
            </button>
            <button
              type="button"
              disabled={isActiveSession(session.status)}
              aria-label={
                isActiveSession(session.status)
                  ? `Delete ${session.display_name} blocked by active run`
                  : `Delete ${session.display_name}`
              }
            >
              Delete
            </button>
          </div>
        </article>
      ))}
    </section>
  );
}

function isActiveSession(status: SessionStatus): boolean {
  return !["draft", "completed", "failed", "terminated"].includes(status);
}

function formatStatus(status: SessionStatus): string {
  const labels: Record<SessionStatus, string> = {
    draft: "Draft",
    running: "Running",
    paused: "Paused",
    waiting_clarification: "Waiting clarification",
    waiting_approval: "Waiting approval",
    waiting_tool_confirmation: "Waiting tool confirmation",
    completed: "Completed",
    failed: "Failed",
    terminated: "Terminated",
  };
  return labels[status];
}
```

Add styling for the sidebar, status density, buttons, and truncation. Keep button heights stable and use standard controls.
