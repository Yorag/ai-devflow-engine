Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add frozen-snapshot Provider retry, exponential backoff, circuit breaker, and StageArtifact process trace recording for model calls without changing run state orchestration or projection owner code.

**Architecture:** Implement a pure retry policy module under `backend/app/providers/` that consumes the existing run-frozen `ProviderCallPolicySnapshotRead`, then add `LangChainProviderAdapter.invoke_with_retry()` as a bounded wrapper around the existing structured invocation path. The adapter returns strict retry and circuit trace records; the runtime caller persists those records through the existing `ArtifactStore.append_process_record()` boundary by writing `provider_retry_trace`, `provider_retry_trace_ref(s)`, `provider_circuit_breaker_trace`, and `provider_circuit_breaker_trace_ref(s)`. A4.9e does not add ArtifactStore helper methods because `backend/app/services/artifacts.py` is AL01 owner scope.

**Tech Stack:** Python 3.11+, Pydantic v2, LangChain adapter code already present in A4.9, repository-local `uv`, and pytest via `uv run pytest`.

Design rules:

- Retry count is the configured retry count, not total attempts.
- `retry_attempt` is one-based for scheduled retry attempts after a failed call.
- Exponential backoff uses `min(backoff_max_seconds, backoff_base_seconds * 2 ** (retry_attempt - 1))`.
- Timeout and network errors use `network_error_max_retries`.
- Rate limit errors use `rate_limit_max_retries`.
- Auth failure, model not found, capability unsupported, model binding/snapshot unavailable, empty response, and unparseable structured output are not retryable.
- Circuit breaker opens when consecutive failed attempts for the same `provider_snapshot_id + model_binding_snapshot_id` reach the frozen threshold.
- Circuit recovery uses only `circuit_breaker_recovery_seconds` from the frozen snapshot.
