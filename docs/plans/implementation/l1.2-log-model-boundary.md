> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task in the main agent session. Steps use checkbox (`- [ ]`) syntax for tracking. Do not use subagent handoff, Git write actions, PR creation, merge, tag, or branch cleanup for this repository slice.

**Goal:** Establish the first `log.db` SQLAlchemy ORM model boundary for run log indexes, audit ledger records, and bounded payload summaries without creating log writers, audit services, query APIs, product projections, runtime behavior, or frontend UI.

**Architecture:** Add a focused `backend/app/db/models/log.py` module whose declarative base is bound to `ROLE_METADATA[DatabaseRole.LOG]`. `RunLogEntryModel` indexes JSONL run log file locations, trace identifiers, product object references, severity/category, and bounded payload references; `AuditLogEntryModel` records audit actor/action/target/result facts with request and correlation identifiers; `LogPayloadModel` stores bounded structured summaries and storage refs, not raw unbounded payload content. Alembic imports the log model module so log role metadata is populated while control, runtime, graph, and event metadata remain isolated.

**Tech Stack:** Python 3.11, SQLAlchemy 2.x, Alembic, pytest through the repo-local `uv run` environment.

## Log & Audit Integration

This slice defines the log/audit persistence boundary itself. It does not write logs or audit records from user commands, runtime nodes, model calls, tools, workspace writes, `bash`, Git delivery, remote delivery, or configuration changes.

Boundary decisions for later integration:

- `RunLogEntryModel` stores JSONL file location fields and shared trace identifiers so later readers can map from index rows to `.runtime/logs/*.jsonl`.
- `AuditLogEntryModel` stores required actor/action/target/result and request/correlation identifiers. The module exposes `AUDIT_WRITE_FAILURE_BEHAVIOR = "reject_or_rollback_high_impact_action"` so later command slices cannot downgrade audit ledger failure to ordinary run-log failure.
- `LogPayloadModel` stores structured summaries, content hash, storage ref, size, and redaction state only. It does not store raw full payload text.
- Tests must prove `log.db` does not contain domain events, Narrative Feed tables, Inspector projections, runtime product state, graph state, or control-plane entities.

RUN_LOG_INDEX_FAILURE_BEHAVIOR = "diagnostic_error_without_domain_rollback"


def _contract_enum(enum_type: type, name: str) -> SqlEnum:
    return SqlEnum(
        enum_type,
        values_callable=lambda values: [item.value for item in values],
        native_enum=False,
        create_constraint=True,
        validate_strings=True,
        name=name,
    )


class LogBase(DeclarativeBase):
    metadata = ROLE_METADATA[DatabaseRole.LOG]


class LogPayloadModel(LogBase):
    __tablename__ = "log_payloads"

    payload_id: Mapped[str] = mapped_column(String(80), primary_key=True)
    payload_type: Mapped[str] = mapped_column(String(80), nullable=False, index=True)
    summary: Mapped[JsonObject] = mapped_column(JSON, nullable=False)
    storage_ref: Mapped[str | None] = mapped_column(String(240), nullable=True)
    content_hash: Mapped[str] = mapped_column(String(160), nullable=False)
    redaction_status: Mapped[RedactionStatus] = mapped_column(
        _contract_enum(RedactionStatus, "log_payload_redaction_status"),
