**Goal:** Implement soft-remove project history commands so a non-default Project without an active run can be removed from product-visible history, cascading the hide to still-visible Session history while preserving runtime facts, audit facts, repository state, and physical project files.

**Architecture:** Extend `ProjectService` with a focused remove command that validates project removability, reuses the existing run lifecycle truth for active-run blocking, bulk-hides visible Session rows and the Project row inside the control database, and records stable audit outcomes for success, blocked removal, missing Project, repeated removal, and default-Project rejection. Expose the command through `DELETE /api/projects/{projectId}` and keep `load_project()` usable after a prior removal by reactivating the hidden Project record without resurrecting previously hidden Session history.

**Tech Stack:** FastAPI, SQLAlchemy 2.x, Pydantic v2, pytest, repository-local `uv run`.

Removal behavior:
- `assert_project_removable(project_id=..., trace_context=...)` loads the Project row without the visible-only list filter so repeated removal can be distinguished from missing Project.
- If the Project row does not exist, raise `ProjectServiceError(ErrorCode.NOT_FOUND, PROJECT_NOT_FOUND_MESSAGE, 404)` and record rejected audit `project.remove.rejected`.
- If the Project exists but `is_visible` is already `False`, raise `ProjectServiceError(ErrorCode.VALIDATION_ERROR, PROJECT_ALREADY_REMOVED_MESSAGE, 409)` and record rejected audit `project.remove.rejected`.
- If the Project is the default Project, raise `ProjectServiceError(ErrorCode.VALIDATION_ERROR, DEFAULT_PROJECT_REMOVE_BLOCKED_MESSAGE, 409)` and record rejected audit `project.remove.rejected`.
- `assert_project_removable()` returns the checked Project row plus the ordered snapshot of currently visible Session rows under that Project.
- Ordered Session snapshot uses the same ordering as existing session lists: `updated_at DESC`, `created_at DESC`, `session_id ASC`.
- Active-run blocker discovery must check the checked visible Sessions in that stable order.
- `remove_project()` requires `runtime_session` for the whole command because it acquires a runtime-side write barrier and performs a project-wide runtime scan before finalizing visibility updates.
- For each checked Session:
  - if a pending startup publication exists, treat it as an active-run blocker and use the pending publication `run_id` as `blocking_run_id`;
  - else if `current_run_id is None`, continue to the next checked Session;
  - else load the `PipelineRunModel`, and if it exists with `RunStateMachine.is_active_run_status(run.status) is True`, block removal with that `run_id`;
  - else if the runtime row is missing, continue the shared blocker scan without inventing a second status rule.
- After the checked visible Session scan, the command performs a project-wide runtime scan under the runtime-side write barrier so an active run that exists without a refreshed `current_run_id` tail still blocks removal.
- `hide_project_sessions()` bulk-hides only the checked still-visible Session rows for this Project. Already-hidden Sessions remain hidden and keep their old `visibility_removed_at`.
- `remove_project()` acquires the runtime-side write barrier, bulk-hides visible Session rows first, then hides the Project row, rechecks runtime blockers while the barrier is still held, commits the control visibility updates, and only then records the success audit with compensation on pre-ledger audit failure.
- `remove_project()` must retry once on guard loss:
  - if checked Session state changes before the visibility update;
  - if a pending startup publication appears for a checked Session after the initial blocker scan;
  - if a new visible Session appears under the Project after the initial blocker scan.
- If recheck after guard loss reveals a pending startup publication or an active run, return the structured blocked result rather than raising.
- If recheck still finds no blocker and the second guarded write loses the race again, raise `ProjectServiceError(ErrorCode.VALIDATION_ERROR, "Project remove state changed before visibility update.", 409)`.
- Successful removal sets:
  - `ProjectModel.is_visible = False`
  - `ProjectModel.visibility_removed_at = now`
  - `ProjectModel.updated_at = now`
  - every checked still-visible `SessionModel.is_visible = False`
  - every checked still-visible `SessionModel.visibility_removed_at = now`
  - every checked still-visible `SessionModel.updated_at = now`
- Successful removal returns:

```python
ProjectRemoveResult(
    project_id=model.project_id,
    visibility_removed=True,
    blocked_by_active_run=False,
    blocking_run_id=None,
    error_code=None,
    message="Project removed from regular product history.",
)
```

- Active-run blocked removal does not mutate the Project or any Session row and returns:

```python
ProjectRemoveResult(
    project_id=model.project_id,
    visibility_removed=False,
    blocked_by_active_run=True,
    blocking_run_id=blocking_run_id,
    error_code="project_active_run_blocks_remove",
    message="Project has an active run.",
)
```

- Project removal must not clear `current_run_id`, must not change Session status, and must not touch runtime/event/log domain rows.
- Project removal must not delete the default delivery channel row or mutate project configuration fields beyond visibility markers.
- `load_project()` must reactivate an existing hidden Project row by setting `is_visible=True`, `visibility_removed_at=None`, and `updated_at=now`, then reusing the existing load audit path.
- Reloading a previously removed Project does not resurrect previously hidden Session rows. `C2.9b` only guarantees the Project context is loadable again.

```python
def build_settings(tmp_path: Path) -> EnvironmentSettings:
    default_root = tmp_path / "platform"

    return EnvironmentSettings(
        platform_runtime_root=tmp_path / "runtime",
        default_project_root=default_root,
```

```python
def test_remove_project_rejects_default_missing_and_already_removed_project(
    tmp_path: Path,
) -> None:
    from backend.app.services.projects import ProjectService, ProjectServiceError

    manager = build_manager(tmp_path)
    default_root = tmp_path / "platform"
    extra_root = tmp_path / "extra-project"
    default_root.mkdir(exist_ok=True)
    extra_root.mkdir()
    seed_project(manager, project_id="project-default", root_path=default_root, is_default=True)
    seed_project(
        manager,
        project_id="project-hidden",
        root_path=extra_root,
        is_visible=False,
        removed_at=NOW,
    )
    audit = RecordingAuditService()

    with (
        manager.session(DatabaseRole.CONTROL) as control_session,
        manager.session(DatabaseRole.RUNTIME) as runtime_session,
    ):
        service = ProjectService(
            control_session,
            settings=build_settings(tmp_path),
            runtime_session=runtime_session,
            audit_service=audit,
            now=lambda: LATER,
        )
        with pytest.raises(ProjectServiceError) as default_blocked:
            service.remove_project(project_id="project-default", trace_context=build_trace())
        with pytest.raises(ProjectServiceError) as already_removed:
            service.remove_project(project_id="project-hidden", trace_context=build_trace())
        with pytest.raises(ProjectServiceError) as missing:
            service.remove_project(project_id="project-missing", trace_context=build_trace())

    assert default_blocked.value.status_code == 409
    assert default_blocked.value.message == "Default Project cannot be removed."
    assert already_removed.value.status_code == 409
    assert already_removed.value.message == "Project was already removed from regular product history."
```

```python
    def _find_blocking_project_run(
        self,
        *,
        checked_sessions: list[CheckedProjectSession],
    ) -> str | None:
        pending_by_session = {
            row.pending_session_id: row.run_id
            for row in self._session.query(StartupPublicationModel)

                StartupPublicationModel.pending_session_id.in_(
                    [session.session_id for session in checked_sessions]

            .all()
```

```python
    def hide_project_sessions(
        self,
        *,
        project_id: str,
        checked_sessions: list[CheckedProjectSession],
        timestamp: datetime,
    ) -> int:
        if not checked_sessions:
            return 0
        guarded_rows = [
            and_(
                SessionModel.session_id == checked.session_id,
                SessionModel.project_id == project_id,
                SessionModel.is_visible.is_(True),
                SessionModel.status == checked.status,
                SessionModel.updated_at == checked.updated_at,
                self._current_run_matches_checked_state(checked.current_run_id),
                self._no_pending_startup_publication(checked.session_id),
            )
            for checked in checked_sessions
        ]
        result = self._session.execute(
            update(SessionModel)
            .where(or_(*guarded_rows))
            .values(
                is_visible=False,
                visibility_removed_at=timestamp,
                updated_at=timestamp,
            )
            .execution_options(synchronize_session=False)
        )
        return int(result.rowcount or 0)
```

```python
    def _current_run_matches_checked_state(self, current_run_id: str | None):  # noqa: ANN202
        if current_run_id is None:
            return SessionModel.current_run_id.is_(None)
        return SessionModel.current_run_id == current_run_id

    def _no_pending_startup_publication(self, session_id: str):  # noqa: ANN202
        return ~exists().where(
            StartupPublicationModel.pending_session_id == session_id,
            StartupPublicationModel.publication_state == "pending",
        )

    def _no_unchecked_visible_project_sessions(
        self,
        project_id: str,
        checked_session_ids: list[str],
    ):  # noqa: ANN202
        return ~exists().where(
            SessionModel.project_id == project_id,
            SessionModel.is_visible.is_(True),
            SessionModel.session_id.not_in(checked_session_ids),
        )
```

```python
    def _soft_remove_checked_project(
        self,
        *,
        model: ProjectModel,
        checked_sessions: list[CheckedProjectSession],
        trace_context: TraceContext,
        retry_on_guard_loss: bool,
    ) -> ProjectRemoveResult:
        timestamp = self._now()
        updated_sessions = self.hide_project_sessions(
            project_id=model.project_id,
            checked_sessions=checked_sessions,
            timestamp=timestamp,
        )
        expected_sessions = len(checked_sessions)
        if updated_sessions != expected_sessions:
            self._session.rollback()
            return self._retry_or_block_project_remove(
                project_id=model.project_id,
                trace_context=trace_context,
                retry_on_guard_loss=retry_on_guard_loss,
            )

        project_result = self._session.execute(
            update(ProjectModel)
            .where(
                ProjectModel.project_id == model.project_id,
                ProjectModel.is_visible.is_(True),
                ProjectModel.is_default.is_(False),
                ProjectModel.updated_at == model.updated_at,
                self._no_unchecked_visible_project_sessions(
                    model.project_id,
                    [checked.session_id for checked in checked_sessions],
                ),
            )
            .values(
                is_visible=False,
                visibility_removed_at=timestamp,
                updated_at=timestamp,
            )
            .execution_options(synchronize_session=False)
        )
        if project_result.rowcount != 1:
            self._session.rollback()
            return self._retry_or_block_project_remove(
                project_id=model.project_id,
                trace_context=trace_context,
                retry_on_guard_loss=retry_on_guard_loss,
            )

        self._session.flush()
        self._record_remove_success_audit(
            model=model,
            hidden_visible_session_count=expected_sessions,
            timestamp=timestamp,
            trace_context=trace_context,
        )
        self._session.commit()
        return ProjectRemoveResult(
            project_id=model.project_id,
            visibility_removed=True,
            blocked_by_active_run=False,
            blocking_run_id=None,
```
