# V6.3 Playwright Control Flow Live Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Fallback to superpowers:executing-plans only when subagents are unavailable, cannot be constrained to the listed files and commands, or the implementation becomes too tightly coupled for safe delegation. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade V6.3 from route-fixture `mock_ready` coverage toward live backend-backed Playwright coverage for manual intervention controls, and stop with precise owner evidence if live completion exposes a non-QA-E2E contract defect.

**Architecture:** Keep default Playwright behavior fixture-backed and add an opt-in live backend mode through `E2E_LIVE_BACKEND=1`. In live mode Playwright starts the test-only FastAPI app from `backend.app.testing.create_e2e_test_app`, points Vite at that backend with `VITE_API_BASE_URL`, and uses hidden deterministic advancement only as a test driver. UI assertions consume normal REST projections, SSE frames, and browser state.

**Tech Stack:** Playwright Test, Vite, FastAPI, uvicorn, repo-local `uv`, deterministic runtime E2E advancement harness.
