## Behavior Details

`ProviderWriteRequest`:
- `display_name: str | None`
- `protocol_type: ProviderProtocolType | None`
- `base_url: str`
- `api_key_ref: str | None`
- `default_model_id: str`
- `supported_model_ids: list[str]`
- `runtime_capabilities: list[ProviderModelRuntimeCapabilitiesWrite]`

`ProviderModelRuntimeCapabilitiesWrite`:
- `model_id: str`
- `context_window_tokens: PositiveInt = 128000`
- `max_output_tokens: PositiveInt | None = None`
- `supports_tool_calling: bool = False`
- `supports_structured_output: bool = False`
- `supports_native_reasoning: bool = False`

Validation rules:
- `custom` Provider creation requires non-empty `display_name`.
- `custom` Provider protocol is always `openai_completions_compatible`; a different submitted protocol is rejected.
- Built-in Provider patch rejects changed `display_name` or changed `protocol_type`.
- `default_model_id` must be present in `supported_model_ids`.
- `runtime_capabilities` must cover every supported model id.
- `context_window_tokens` and `max_output_tokens` must be positive integers after defaults are applied.
- `api_key_ref` must be `None` or an `env:<NAME>` reference; raw key-looking values are rejected as config values and are not saved or audited.
- Saved `runtime_capabilities` must be normalized through `ProviderService.apply_model_capability_defaults()` so missing `max_output_tokens` becomes `4096` and missing booleans become `False`.
- Provider writes must not modify run snapshots, `ProviderSnapshot`, `ModelBindingSnapshot`, templates, DeliveryChannel, configuration packages, or environment settings.

Service behavior:
- `get_provider(provider_id, trace_context)` seeds missing built-ins through the existing seed path, then returns the Provider row or `None`.
- `create_custom_provider(body, trace_context)` creates `ProviderModel(provider_source=custom, protocol_type=openai_completions_compatible)` with `provider_id = provider-custom-<uuid hex>`, audits `provider.create_custom`, and commits only after audit succeeds.
- `patch_custom_provider(provider_id, body, trace_context)` updates display name when provided plus connection fields, model list, credential ref, and capabilities; audits `provider.patch_custom`; and commits only after audit succeeds.
- `patch_builtin_provider_runtime_config(provider_id, body, trace_context)` preserves built-in identity fields and only updates runtime config fields; audits `provider.patch_builtin_runtime_config`; and commits only after audit succeeds.
- `patch_provider(provider_id, body, trace_context)` dispatches to the built-in or custom patch path after loading the row.
- Missing Provider returns `not_found`, 404.
- Immutable built-in identity mutation returns `validation_error`, 409.
- Invalid Provider configuration returns `config_invalid_value`, 422.

Error messages:
- Missing Provider: `Provider was not found.`
- Custom Provider missing display name: `Custom Provider display_name is required.`
- Custom Provider protocol mismatch: `Custom Provider protocol_type must be openai_completions_compatible.`
- Built-in identity change: `Built-in Provider identity fields cannot be modified.`
- Invalid model binding: `Provider default_model_id must be in supported_model_ids.`
- Missing model capability: `Provider runtime_capabilities must cover supported_model_ids.`
- Invalid credential reference: `Provider api_key_ref must use an env: credential reference.`

Log & Audit Integration:
- Runtime log category: not applicable; C2.5 writes control-plane configuration and does not create run logs.
- Audit actions:
  - Success: `provider.create_custom`, `provider.patch_custom`, `provider.patch_builtin_runtime_config`.
  - Rejected: `provider.create_custom.rejected`, `provider.patch_custom.rejected`, `provider.patch_builtin_runtime_config.rejected`, `provider.patch.rejected`.
- Actor: `AuditActorType.USER`, `actor_id="api-user"` for API/control-plane commands.
- Target: `target_type="provider"`, `target_id` is the created/existing/requested Provider id.
- Trace fields: all route calls pass `get_trace_context()` into service methods. Audit records inherit `request_id`, `trace_id`, `correlation_id`, `span_id`, and `parent_span_id` through `AuditService`.
- Redaction: audit metadata includes ids, enum values, model ids, counts, capability booleans, and credential references only. It must not include raw API key values, Authorization headers, cookies, or prompt text.
- Audit failure behavior: Provider create and patch are configuration-sensitive control-plane commands. Audit ledger failure must raise and rollback the control write. Tests must prove create and patch do not persist changes when success audit fails.
- Product truth: audit records are security/control-plane facts only; `ProviderModel` remains the Provider configuration truth.

class RecordingAuditService:
    def __init__(self) -> None:
        self.records: list[dict[str, Any]] = []

    def record_command_result(self, **kwargs: Any) -> object:
        self.records.append({"method": "record_command_result", **kwargs})
        return object()

    def record_rejected_command(self, **kwargs: Any) -> object:
        self.records.append(
            {
                "method": "record_rejected_command",
                "result": AuditResult.REJECTED,
                **kwargs,
            }
        )
        return object()


class FailingAuditService:
    def record_command_result(self, **kwargs: Any) -> object:
        raise RuntimeError("audit ledger unavailable")

    def record_rejected_command(self, **kwargs: Any) -> object:
        raise RuntimeError("audit ledger unavailable")


def build_trace() -> TraceContext:
    return TraceContext(
        request_id="request-provider-command",
        trace_id="trace-provider-command",
        correlation_id="correlation-provider-command",
        span_id="span-provider-command",
        parent_span_id=None,
        created_at=NOW,
    )


def build_manager(tmp_path: Path) -> DatabaseManager:
    manager = DatabaseManager(
        _database_paths={role: tmp_path / f"{role.value}.db" for role in DatabaseRole},
        _database_urls={
            role: f"sqlite:///{(tmp_path / f'{role.value}.db').as_posix()}"
            for role in DatabaseRole
        },
    )
    ControlBase.metadata.create_all(manager.engine(DatabaseRole.CONTROL))
    return manager

    provider_id: str,
    *,
    trace_context: TraceContext,
) -> ProviderModel | None:
    self.seed_builtin_providers(trace_context=trace_context)

def create_custom_provider(
    self,
    body: ProviderWriteRequest,
    *,
    trace_context: TraceContext,
) -> ProviderModel:
    try:
        payload = self._validated_write_payload(
            body,
            existing_provider=None,
            is_builtin=False,
            rejected_action="provider.create_custom.rejected",
            target_id="provider-custom-new",
            trace_context=trace_context,
        )
    except Exception:
        self._session.rollback()
        raise

    timestamp = self._now()
    provider = ProviderModel(
        provider_id=self._new_custom_provider_id(),
        display_name=payload["display_name"],
        provider_source=ProviderSource.CUSTOM,
        protocol_type=ProviderProtocolType.OPENAI_COMPLETIONS_COMPATIBLE,
        base_url=payload["base_url"],
        api_key_ref=payload["api_key_ref"],
        default_model_id=payload["default_model_id"],
        supported_model_ids=payload["supported_model_ids"],
        runtime_capabilities=payload["runtime_capabilities"],
        created_at=timestamp,
        updated_at=timestamp,
    )
    self._session.add(provider)
    self._session.flush()
    try:
        self._record_success(
            action="provider.create_custom",
            provider=provider,
            old_api_key_ref=None,
            trace_context=trace_context,
        )
        self._session.commit()
    except Exception:
        self._session.rollback()
        raise
    return provider


def patch_provider(
    self,
    provider_id: str,
    body: ProviderWriteRequest,
    *,
    trace_context: TraceContext,
) -> ProviderModel:
    provider = self.get_provider(provider_id, trace_context=trace_context)
    if provider is None:
        self._record_rejected(
            action="provider.patch.rejected",
            target_id=provider_id,
            message="Provider was not found.",
            error_code=ErrorCode.NOT_FOUND,
            metadata={"provider_id": provider_id},
            trace_context=trace_context,
        )
        raise ProviderServiceError(
            ErrorCode.NOT_FOUND,
            "Provider was not found.",
            404,
        )
    if provider.provider_source is ProviderSource.BUILTIN:
        return self.patch_builtin_provider_runtime_config(
            provider_id,
            body,
            trace_context=trace_context,
        )
    return self.patch_custom_provider(provider_id, body, trace_context=trace_context)
