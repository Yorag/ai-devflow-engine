> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` as the default execution skill for this plan. Use `superpowers:executing-plans` only if bounded `gpt-5.5` subagent dispatch is unavailable or the task boundary cannot be kept inside the allowed files. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the Stage Agent structured decision schema and parser so model output can express tool calls, tool-confirmation requests, stage artifact submission, clarification requests, structured repair, retry plans, and stage failure without relying on free text.

**Architecture:** Add a pure runtime parser module under `backend/app/runtime/` that consumes A4.9 `ModelCallResult`, A4.9a/A4.9b `ContextEnvelope.available_tools`, and the current stage contract mapping. The parser returns strict Pydantic `AgentDecision` objects or raises a structured local parser error; it does not execute tools, mutate run/stage state, create confirmations, write artifacts, persist process records, call providers, or parse natural-language commands.

**Tech Stack:** Python 3.11+, Pydantic v2, existing `ModelCallResult`, existing `ContextEnvelope`, existing `ToolBindableDescription`, repository-local `uv`, and pytest via `uv run pytest`.

## Log And Audit Integration

This slice parses model-call decisions and returns trace-ready data. It does not persist runtime logs or audit records.

Parser responsibilities:

- Include `decision_trace` in successful `AgentDecision` objects with:
  - `trace_ref`
  - `status = "accepted"`
  - `decision_type`
  - `model_call_ref`
  - `provider_snapshot_id`
  - `model_binding_snapshot_id`
  - `model_call_type`
  - `request_id`, `trace_id`, `correlation_id`, `span_id`, `parent_span_id`
  - `run_id`, `stage_run_id`
- Include equivalent `decision_trace` in `AgentDecisionParserError.error` with `status = "rejected"` and a stable reason.
- Treat illegal tool calls, invalid tool-call candidates, invalid stage artifact submissions, missing clarification permission, and parse ambiguity as structured model-call errors. Do not execute tools or create confirmations.
- Keep full raw model content out of parser errors. Errors may include bounded safe fields such as `tool_name`, `call_id`, `decision_type`, `reason`, and `model_call_ref`.
- Do not let decision traces become product truth; Stage Agent Runtime and `ArtifactStore` will decide later how to persist accepted/rejected traces under `StageArtifact.process`.

Design rules:

- Strict Pydantic models use `extra="forbid"`.
- `AgentDecision.decision_type` mirrors the selected payload type.
- `request_tool_call` from native `ModelCallResult.tool_call_requests` wins only when there is exactly one executable native call and no conflicting structured decision.
- Native tool calls must validate:
  - tool name exists in `ContextEnvelope.available_tools`
  - tool name appears in the `stage_contract["allowed_tools"]` sequence when present
  - schema version matches the available tool schema version when the model provided a schema version
  - input payload validates with `ToolInputSchemaValidator`
  - duplicate call ids or multiple native tool calls are rejected for this parser slice
- `request_tool_confirmation` is a structured decision only. The parser validates required summary, target, risk, side-effect, and alternative-path fields, then returns data for A4.9d to hand off later; it never creates a `ToolConfirmationRequest`.
- `submit_stage_artifact` validates:
  - `artifact_type` matches `stage_contract["structured_artifact_required"]` when present
  - `artifact_payload` is a non-empty JSON object
  - `evidence_refs` is non-empty
  - risk/failure summary fields are structured when present
- `request_clarification` requires `stage_contract["clarification_allowed"] is True` or `stage_contract["can_request_clarification"] is True`.
- `fail_stage` requires a safe user-visible summary, failure reason, evidence refs, and incomplete items.
- The parser does not parse `AIMessage.content`, natural-language summaries, regex commands, or JSON text strings. It only consumes already structured mappings and normalized native tool-call candidates from `ModelCallResult`.

class AgentDecision(_StrictBaseModel):
    decision_type: AgentDecisionType
    decision_trace: AgentDecisionTrace
    tool_call: ToolCallDecision | None = None
    tool_confirmation: ToolConfirmationDecision | None = None
    stage_artifact: SubmitStageArtifactDecision | None = None
    clarification: ClarificationDecision | None = None
    structured_repair: StructuredRepairDecision | None = None
    retry: RetryWithRevisedPlanDecision | None = None
    fail_stage: FailStageDecision | None = None

    @model_validator(mode="after")
    def validate_single_payload(self) -> Self:
        payloads = {
            AgentDecisionType.REQUEST_TOOL_CALL: self.tool_call,
            AgentDecisionType.REQUEST_TOOL_CONFIRMATION: self.tool_confirmation,
            AgentDecisionType.SUBMIT_STAGE_ARTIFACT: self.stage_artifact,
            AgentDecisionType.REQUEST_CLARIFICATION: self.clarification,
            AgentDecisionType.REPAIR_STRUCTURED_OUTPUT: self.structured_repair,
            AgentDecisionType.RETRY_WITH_REVISED_PLAN: self.retry,
            AgentDecisionType.FAIL_STAGE: self.fail_stage,
        }
        if payloads[self.decision_type] is None:
            raise ValueError("decision payload must match decision_type")
        extra_payloads = [
            decision_type.value
            for decision_type, payload in payloads.items()
            if decision_type is not self.decision_type and payload is not None
        ]
        if extra_payloads:
            raise ValueError("only one decision payload may be set")
        return self

```python
    def validate_against_stage_contract(
        self,
        decision: AgentDecision,
        *,
        context_envelope: ContextEnvelope,
        stage_contract: dict[str, object],
    ) -> AgentDecision:
        if decision.tool_call is not None:
            self._validate_tool_call(
                decision.tool_call,
                model_result_context=decision.decision_trace,
                context_envelope=context_envelope,
                stage_contract=stage_contract,
            )
        if decision.stage_artifact is not None:
            expected = stage_contract.get("structured_artifact_required")
            if isinstance(expected, str) and expected:
                if decision.stage_artifact.artifact_type != expected:
                    self._raise_error_from_trace(
                        AgentDecisionErrorCode.STAGE_CONTRACT_VIOLATION,
                        "Stage artifact type does not match the stage contract.",
                        trace=decision.decision_trace,
                        safe_details={
                            "artifact_type": decision.stage_artifact.artifact_type,
                            "expected_artifact_type": expected,
                        },
                    )
        if decision.clarification is not None:
            allowed = (
                stage_contract.get("clarification_allowed") is True
                or stage_contract.get("can_request_clarification") is True
            )
            if not allowed:
                self._raise_error_from_trace(
                    AgentDecisionErrorCode.CLARIFICATION_NOT_ALLOWED,
                    "The current stage contract does not allow clarification requests.",
                    trace=decision.decision_trace,
                    safe_details={},
                )
        return decision

    def _validate_tool_call(
        self,
        decision: ToolCallDecision,
        *,
        model_result_context: AgentDecisionTrace,
        context_envelope: ContextEnvelope,
        stage_contract: dict[str, object],
    ) -> None:
        available = {tool.name: tool for tool in context_envelope.available_tools}
        tool = available.get(decision.tool_name)
        if tool is None:
            self._raise_error_from_trace(
                AgentDecisionErrorCode.TOOL_NOT_AVAILABLE,
                "Tool call references a tool that is not available in the current context.",
                trace=model_result_context,
                safe_details={"tool_name": decision.tool_name},
            )
        allowed_tools = stage_contract.get("allowed_tools")
        if isinstance(allowed_tools, list) and decision.tool_name not in allowed_tools:
            self._raise_error_from_trace(
                AgentDecisionErrorCode.TOOL_NOT_AVAILABLE,
                "Tool call references a tool that is not allowed for the current stage.",
                trace=model_result_context,
                safe_details={"tool_name": decision.tool_name},
            )
        if decision.schema_version is not None and decision.schema_version != tool.schema_version:
            self._raise_error_from_trace(
                AgentDecisionErrorCode.TOOL_SCHEMA_MISMATCH,
                "Tool call schema version does not match the current available tool schema.",
                trace=model_result_context,
                safe_details={
                    "tool_name": decision.tool_name,
                    "schema_version": decision.schema_version,
                    "expected_schema_version": tool.schema_version,
                },
            )
        try:
            self._tool_input_validator.validate(tool.input_schema, decision.input_payload)
        except ValueError as exc:
            self._raise_error_from_trace(
                AgentDecisionErrorCode.TOOL_INPUT_SCHEMA_INVALID,
                "Tool call input does not match the registered schema.",
                trace=model_result_context,
                safe_details={"tool_name": decision.tool_name, "reason": str(exc)},
            )
```
