# H4.4b Tool Confirmation Deny Follow-Up Source Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist stable runtime-side deny follow-up semantics for tool confirmations so `AL02-Q3.4b` can later project them into top-level `tool_confirmation` payloads without inferring from `alternative_path_summary` or terminal run status.

**Architecture:** Add runtime-owned planned deny source fields plus final deny result fields to `ToolConfirmationRequestModel`. `ToolConfirmationService.create_request()` records the planned deny branch source in internal stable columns, `deny()` validates that source and then materializes the final persisted `deny_followup_action` / `deny_followup_summary`, and focused service/API tests prove the three allowed outcomes plus paused/terminal failure behavior.

**Tech Stack:** Python, SQLAlchemy, FastAPI, Pydantic, pytest

```python
def deny(...):
    return self._execute_command(..., decision=ToolConfirmationStatus.DENIED, reason=reason)

def _execute_command(...):
    ...
    deny_followup_action, deny_followup_summary = self._resolve_deny_followup_source(
        request=request,
        decision=decision,
    )
    ...
    self._runtime_orchestration.resume_tool_confirmation(
        ...,
        resume_payload=RuntimeResumePayload(
            ...,
            values={
                "decision": decision.value,
                "tool_confirmation_id": tool_confirmation_id,
                "confirmation_object_ref": request.confirmation_object_ref,
                **({"reason": reason} if reason is not None else {}),
                **(
                    {
                        "deny_followup_action": deny_followup_action,
                        "deny_followup_summary": deny_followup_summary,
                    }
                    if decision is ToolConfirmationStatus.DENIED
                    else {}
                ),
            },
        ),
        ...
    )
```

```python
def _resolve_deny_followup_source(
    self,
    *,
    request: ToolConfirmationRequestModel,
    decision: ToolConfirmationStatus,
) -> tuple[str | None, str | None]:
    if decision is not ToolConfirmationStatus.DENIED:
        return None, None
    if (
        not request.planned_deny_followup_action
        or not request.planned_deny_followup_summary
    ):
        raise RuntimeError("deny follow-up source is missing")
    if request.planned_deny_followup_action not in {
        "continue_current_stage",
        "run_failed",
        "awaiting_run_control",
    }:
        raise RuntimeError("deny follow-up action is invalid")
    return (
        request.planned_deny_followup_action,
        request.planned_deny_followup_summary,
    )
```

Implementation rules:

- Do not extend `ToolConfirmationFeedEntry`.
- Do not derive the three stable values from `alternative_path_summary` or current terminal status.
- Keep final `deny_followup_action` / `deny_followup_summary` as `None` for non-denied states.
- Preserve existing rollback behavior: any failure before `_commit_all()` must leave the request unresolved.
