# L6.1 Log Rotation And Retention Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the official QA-OBS-L6.1 worker slice for run-log rotation and retention cleanup on coordination base `2e682ec`.

**Architecture:** Introduce `LogRetentionService` as a standalone observability service that resolves runtime-relative log refs under `.runtime/logs`, rotates oversized or previous-day JSONL files, updates run-log index refs for rotated files, cleans expired non-protected run-log index rows, and deletes only fully unreferenced local run log files. Audit entries and high-impact run-log records use separate retention boundaries and are never deleted by the ordinary run-log cleanup path.

**Tech Stack:** Python 3.11+, pathlib, SQLAlchemy ORM sessions, existing `RuntimeDataSettings`, existing `RunLogEntryModel`, `LogPayloadModel`, and `AuditLogEntryModel`.

### Task 3: Retention Service Implementation

**Files:**
- Create: `backend/app/observability/retention.py`
- Test: `backend/tests/observability/test_log_retention.py`

- [ ] **Step 1: Implement service result dataclasses**

Add immutable result objects for rotation and cleanup:

```python
@dataclass(frozen=True)
class LogRotationResult:
    rotated: bool
    original_log_file_ref: str
    rotated_log_file_ref: str | None
    log_file_generation: str | None
    reason: str | None


@dataclass(frozen=True)
class LogCleanupResult:
    cutoff: datetime
    deleted_log_ids: tuple[str, ...]
    deleted_payload_refs: tuple[str, ...]
    deleted_file_refs: tuple[str, ...]
    retained_file_refs: tuple[str, ...]
    protected_log_ids: tuple[str, ...]
```

- [ ] **Step 2: Implement runtime-relative path resolution**

Resolve log refs only under `RuntimeDataSettings.root`, reject absolute refs, Windows drive refs, empty segments, `.`, and `..`, and require cleanup file deletion to stay under `RuntimeDataSettings.run_logs_dir`.

- [ ] **Step 3: Implement `rotate_if_needed()`**

Rotate when file size is greater than or equal to `max_bytes`, or when the file mtime UTC date is older than `now.date()`. Use sortable UTC timestamps in rotated names, for example:

```text
logs/runs/run-1.20260501T090000Z.jsonl
```

When a log session is present, update matching `RunLogEntryModel.log_file_ref` and `RunLogEntryModel.log_file_generation` while preserving `line_offset` and `line_number`.

- [ ] **Step 4: Implement `cleanup_run_logs()`**

Delete only expired ordinary run-log index rows older than `now - retention_days`. Preserve rows with `LogCategory.SECURITY`, `approval_id`, `tool_confirmation_id`, or `delivery_record_id`. Delete a local run-log JSONL file only when every indexed row that references that file is being removed and the path is under `logs/runs/`. Do not delete audit rows, audit payload refs, runtime rows, event rows, control rows, or delivery records.

- [ ] **Step 5: Implement `mark_log_expired()`**

Update a single run-log row to an explicit expired sentinel using existing columns. Preserve row identity and trace fields so query projection remains stable.
