## Behavior Details

`ProjectDeliveryChannelUpdateRequest`:

```python
class ProjectDeliveryChannelUpdateRequest(_StrictBaseModel):
    delivery_mode: common.DeliveryMode
    scm_provider_type: common.ScmProviderType | None = None
    repository_identifier: str | None = None
    default_branch: str | None = None
    code_review_request_type: common.CodeReviewRequestType | None = None
    credential_ref: str | None = None
```

The request schema only parses submitted fields and forbids extras. Mode-dependent
required-field checks, whitespace normalization, credential-ref validation, and
rejected audit records are handled inside `DeliveryChannelService.update_project_channel()`
so invalid C2.6 configuration attempts still inherit L2.1 trace context and L2.4
audit behavior.

Service behavior:
- `get_project_channel(project_id, trace_context)` loads a visible Project, then loads `project.default_delivery_channel_id`.
- `get_project_channel()` does not call `ensure_default_channel()` and does not create a second channel on read.
- Missing Project returns `not_found`, 404, `Project was not found.`
- Missing default channel returns `not_found`, 404, `DeliveryChannel was not found.`
- `update_project_channel(project_id, body, trace_context)` loads the existing default channel, applies the update, records `delivery_channel.save`, and commits only after audit succeeds.
- `demo_delivery` save clears all Git fields, sets `credential_ref = None`, `credential_status = ready`, `readiness_status = ready`, `readiness_message = None`, and `last_validated_at = None`.
- `git_auto_delivery` save writes all submitted Git fields and `credential_ref`, sets `credential_status = unbound`, `readiness_status = unconfigured`, `readiness_message = "DeliveryChannel readiness has not been validated."`, and `last_validated_at = None`. C2.7 will compute ready/invalid states.
- `git_auto_delivery` missing or blank Git fields are rejected by the service as `config_invalid_value`, audited with `delivery_channel.save.rejected`, and not persisted.
- Valid padded `repository_identifier` and `default_branch` strings are stripped before saving.
- `demo_delivery` accepts stale Git fields from a settings form and clears them without validating stale `credential_ref`.
- `update_project_channel()` requires an audit service before persistence; `ensure_default_channel()` remains compatible with C2.1 default-channel creation.
- Invalid credential reference returns `config_invalid_value`, 422, `DeliveryChannel credential_ref must use an env: credential reference.`
- Save rejections audit `delivery_channel.save.rejected`.
- Save success metadata includes enum values, project id, delivery channel id, repository target fields, and a `ref_transition` for credential reference changes. It does not include raw credentials; unsafe legacy credential references are represented as `[blocked:credential_ref]`.

Error messages:
- Missing Project: `Project was not found.`
- Missing DeliveryChannel: `DeliveryChannel was not found.`
- Invalid credential reference: `DeliveryChannel credential_ref must use an env: credential reference.`

## Log & Audit Integration

- Runtime log category: not applicable; C2.6 writes control-plane configuration and does not create run logs.
- Audit actions:
  - Success: `delivery_channel.save`.
  - Rejected: `delivery_channel.save.rejected`.
- Actor: `AuditActorType.USER`, `actor_id="api-user"`.
- Target: `target_type="delivery_channel"`, `target_id` is the resolved channel id, or `project:<project_id>` when no channel can be resolved.
- Trace fields: route calls pass `get_trace_context()` into the service. Audit records inherit `request_id`, `trace_id`, `correlation_id`, `span_id`, and `parent_span_id` through `AuditService`.
- Redaction: audit metadata contains ids, enum values, repository identifiers, default branch, code review request type, and credential references only. It must not include raw credential values, Authorization headers, cookies, API keys, private keys, prompt text, or run snapshot refs.
- Audit failure behavior: success audit failure raises and rolls back the control write. Tests prove that a failed audit does not persist a delivery channel update.
- Product truth: audit records are security/control-plane facts only; `DeliveryChannelModel` remains the current Project delivery configuration truth.

```python
    def __init__(self) -> None:
        self.records: list[dict[str, Any]] = []

        self.records.append({"method": "record_command_result", **kwargs})

        self.records.append(
```

```python
def get_project_channel(
    self,
    project_id: str,
    *,
    trace_context: TraceContext,
) -> DeliveryChannelModel:
    project = self._load_project(project_id)
    return self._load_default_channel(project)


def update_project_channel(
    self,
    project_id: str,
    body: ProjectDeliveryChannelUpdateRequest,
    *,
    trace_context: TraceContext,
) -> DeliveryChannelModel:
    try:
        project = self._load_project(project_id)
        channel = self._load_default_channel(project)
        old_credential_ref = channel.credential_ref
        self._validate_credential_ref(
            body.credential_ref,
            target_id=channel.delivery_channel_id,
            trace_context=trace_context,
        )
        self._apply_update(channel, body)
        channel.updated_at = self._now()
        self._session.add(channel)
        self._session.flush()
        self._record_success(
            channel=channel,
            old_credential_ref=old_credential_ref,
            trace_context=trace_context,
        )
        self._session.commit()
        return channel
    except Exception:
        self._session.rollback()
        raise
```

Implementation details:
- `_load_project()` queries `ProjectModel` by `project_id` and `is_visible.is_(True)`.
- `_load_default_channel()` loads `ProjectModel.default_delivery_channel_id`; it must not call `ensure_default_channel()`.
- `_apply_update()` clears Git fields and sets ready statuses for demo mode; for git mode it writes all request fields and unvalidated statuses.
- `_validate_credential_ref()` accepts `None` or `env:<NAME>` where `<NAME>` is non-empty, ASCII alphanumeric plus underscores, and starts with an allowed prefix.
- `_record_success()` calls `record_command_result()` with `delivery_channel.save`.
- `_record_rejected()` calls `record_rejected_command()` with `delivery_channel.save.rejected`.
- Rejected metadata for invalid credentials must only include `credential_ref_status = "invalid_reference"` and `error_code`, not the raw submitted value.

```python
def build_delivery_channel_api_app(tmp_path: Path):
    default_root = tmp_path / "ai-devflow-engine"
    default_root.mkdir()
    settings = EnvironmentSettings(
        platform_runtime_root=tmp_path / "runtime",
        default_project_root=default_root,
    )
    app = create_app(settings=settings)
    ControlBase.metadata.create_all(app.state.database_manager.engine(DatabaseRole.CONTROL))
    LogBase.metadata.create_all(app.state.database_manager.engine(DatabaseRole.LOG))
    return app


def git_payload(
    *,
    credential_ref: str | None = "env:AI_DEVFLOW_CREDENTIAL_DELIVERY_TOKEN",
) -> dict:
    return {
        "delivery_mode": "git_auto_delivery",

        "repository_identifier": "acme/app",
        "default_branch": "main",
        "code_review_request_type": "pull_request",
        "credential_ref": credential_ref,
    }


def assert_error(

    *,
    status_code: int,
    error_code: str,
    message: str,
    request_id: str,
    correlation_id: str,
) -> None:
    assert response.status_code == status_code
    assert response.json() == {
        "error_code": error_code,
        "message": message,
        "request_id": request_id,
        "correlation_id": correlation_id,
    }
```

```python
def test_project_delivery_channel_routes_return_unified_errors_and_rejected_audit(
    tmp_path: Path,
) -> None:
    app = build_delivery_channel_api_app(tmp_path)

    with TestClient(app) as client:
        missing_project = client.get(
            "/api/projects/project-missing/delivery-channel",
            headers={
                "X-Request-ID": "req-delivery-missing",
                "X-Correlation-ID": "corr-delivery-missing",
            },
        )
        raw_key = client.put(
            "/api/projects/project-default/delivery-channel",
            json=git_payload(credential_ref="raw-secret-value"),
            headers={
                "X-Request-ID": "req-delivery-raw-key",
                "X-Correlation-ID": "corr-delivery-raw-key",
            },
        )
        missing_git_field = client.put(
            "/api/projects/project-default/delivery-channel",
            json={
                "delivery_mode": "git_auto_delivery",
                "scm_provider_type": "github",
                "repository_identifier": "acme/app",
                "default_branch": "main",
                "credential_ref": "env:AI_DEVFLOW_CREDENTIAL_DELIVERY_TOKEN",
            },
            headers={
                "X-Request-ID": "req-delivery-shape",
                "X-Correlation-ID": "corr-delivery-shape",
            },
        )

    assert_error(
        missing_project,
        status_code=404,
        error_code="not_found",
        message="Project was not found.",
        request_id="req-delivery-missing",
        correlation_id="corr-delivery-missing",
    )
    assert_error(
        raw_key,
        status_code=422,
        error_code="config_invalid_value",
        message="DeliveryChannel credential_ref must use an env: credential reference.",
        request_id="req-delivery-raw-key",
        correlation_id="corr-delivery-raw-key",
    )
    assert_error(
        missing_git_field,
        status_code=422,
        error_code="config_invalid_value",
        message="git_auto_delivery requires code_review_request_type",
        request_id="req-delivery-shape",
        correlation_id="corr-delivery-shape",
    )

    with app.state.database_manager.session(DatabaseRole.LOG) as session:
        rejected = (
            session.query(AuditLogEntryModel)
            .filter(
                AuditLogEntryModel.action == "delivery_channel.save.rejected",
                AuditLogEntryModel.request_id == "req-delivery-raw-key",
            )
            .one_or_none()
        )
    assert rejected is not None
    assert rejected.result is AuditResult.REJECTED
    assert "raw-secret-value" not in (rejected.metadata_excerpt or "")
```
