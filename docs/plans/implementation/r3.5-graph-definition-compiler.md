**Goal:** Compile a frozen `TemplateSnapshot` plus `RuntimeLimitSnapshot` into a stable six-stage `GraphDefinition` that fixes stage contracts, approval interrupts, retry routing, delivery routing, and node-group-to-stage mapping for later runtime consumption.

**Architecture:** This slice introduces a pure domain `GraphDefinition` model and a focused `GraphCompiler` service. The compiler derives fixed stage nodes and stage contracts from current specs, stores run-snapshot provenance inside the compiled definition, and emits log-summary callbacks for compile success or failure without becoming the run-start transaction owner. This slice does not wire first-run startup, persist graph definitions into `graph.db`, create `GraphThread`, modify API routes, or change runtime orchestration behavior.

**Tech Stack:** Python 3.11+, Pydantic v2, existing domain enums/snapshot models, pytest via `uv run python -m pytest`.

**Execution Status:** Local worker result `reported`; expected committed ingest result `implemented`.

## Design Decisions

- `GraphDefinition` is a frozen Pydantic domain model, not a SQLAlchemy model and not a LangGraph compiled graph wrapper.
- The compiler output must encode:
  - six formal business stages in the fixed order from `TemplateSnapshot.fixed_stage_sequence`
  - stable graph node keys
  - internal node-group metadata for `solution_design` validation and code-review retry routing
  - `source_node_group -> stage_type` mapping
  - `interrupt_policy`
  - `retry_policy`
  - `delivery_routing_policy`
  - stage contracts for all six stages
  - `runtime_limit_snapshot_ref`
  - `runtime_limit_source_config_version`
- Because `GraphDefinitionModel` from `C1.8` does not yet expose dedicated columns for `runtime_limit_snapshot_ref` or runtime-settings provenance, this slice stores those values in the domain object and inside contract/policy payloads only. No graph-db schema change is part of this claim.
- `solution_design_approval` and `code_review_approval` remain fixed approval interrupt identifiers in compile output.
- `Solution Validation` is represented as an internal node group in the `solution_design` stage metadata and never appears as a separate stage node or stage type.
- `allowed_tools` are fixed from current backend specs, not from template data:
  - `requirement_analysis`: `[]`
  - `solution_design`: `["read_file", "glob", "grep"]`
  - `code_generation`: `["read_file", "glob", "grep", "write_file", "edit_file"]`
  - `test_generation_execution`: `["read_file", "glob", "grep", "write_file", "edit_file", "bash"]`
  - `code_review`: `["read_file", "glob", "grep"]`
  - `delivery_integration`: `["read_delivery_snapshot", "prepare_branch", "create_commit", "push_branch", "create_code_review_request"]`
- `requirement_analysis` remains tool-free by default in the compiled contract for this slice. Spec language allows read-only tools “when needed”, but the stable minimal V1 contract table later fixes the allowed tool set as default `[]`; this claim chooses the stricter fixed contract to avoid a speculative second configuration source.
- Runtime-limit usage in contracts should reference the frozen snapshot by id and embed the per-stage limits the runtime will need later, at minimum:
  - `max_react_iterations_per_stage`
  - `max_tool_calls_per_stage`
  - `max_structured_output_repair_attempts`
  - `max_auto_regression_retries`
  - `max_clarification_rounds`
  - `max_no_progress_iterations`
- Compile validation uses `ErrorCode.VALIDATION_ERROR` for invalid template/run-shape input and missing fixed-stage alignment, because this slice validates graph compilation inputs rather than runtime-settings storage availability.

```python
from __future__ import annotations

from datetime import datetime
from typing import Any, Literal

from pydantic import BaseModel, ConfigDict, Field, StrictStr, field_validator

from backend.app.domain.enums import StageType


GRAPH_DEFINITION_SCHEMA_VERSION = "graph-definition-v1"
GRAPH_DEFINITION_VERSION = "function-one-mainline-v1"
JsonObject = dict[str, Any]


class GraphDefinition(BaseModel):
    model_config = ConfigDict(extra="forbid", frozen=True)

    graph_definition_id: StrictStr = Field(min_length=1, max_length=80)
    run_id: StrictStr = Field(min_length=1)
    template_snapshot_ref: StrictStr = Field(min_length=1)
    runtime_limit_snapshot_ref: StrictStr = Field(min_length=1)
    runtime_limit_source_config_version: StrictStr = Field(min_length=1)
    graph_version: Literal["function-one-mainline-v1"] = GRAPH_DEFINITION_VERSION
    stage_nodes: tuple[JsonObject, ...]
    stage_contracts: dict[str, JsonObject]
    interrupt_policy: JsonObject
    retry_policy: JsonObject
    delivery_routing_policy: JsonObject
    source_node_group_map: dict[str, str]
    schema_version: Literal["graph-definition-v1"] = GRAPH_DEFINITION_SCHEMA_VERSION
    created_at: datetime

    @field_validator("stage_nodes")
    @classmethod
    def _require_six_stage_nodes(
        cls,
        value: tuple[JsonObject, ...],
    ) -> tuple[JsonObject, ...]:
        if len(value) != 6:
            raise ValueError("stage_nodes must contain the six formal business stages")
        return value

    @field_validator("stage_contracts")
    @classmethod
    def _require_stage_contract_keys(
        cls,
        value: dict[str, JsonObject],
    ) -> dict[str, JsonObject]:
        expected = {stage.value for stage in StageType}
        if set(value) != expected:
            raise ValueError("stage_contracts must cover all formal stage types")
        return value

    @field_validator("source_node_group_map")
    @classmethod
    def _require_stage_type_targets(
        cls,
        value: dict[str, str],
    ) -> dict[str, str]:
        valid_targets = {stage.value for stage in StageType}
        invalid = sorted(target for target in value.values() if target not in valid_targets)
        if invalid:
            raise ValueError("source_node_group_map must target formal stage types only")
        return value
```

```python
    def compile(
        self,
        *,
        template_snapshot: TemplateSnapshot,
        runtime_limit_snapshot: RuntimeLimitSnapshot,
    ) -> GraphDefinition:
        if template_snapshot.run_id != runtime_limit_snapshot.run_id:
            self._record(
                status="failed",
                run_id=runtime_limit_snapshot.run_id,
                error_code=ErrorCode.VALIDATION_ERROR.value,
                reason="run_id_mismatch",
            )
            raise GraphCompilerError(
                ErrorCode.VALIDATION_ERROR,
                "TemplateSnapshot and RuntimeLimitSnapshot run_id must match.",
            )
        if tuple(template_snapshot.fixed_stage_sequence) != build_fixed_stage_sequence():
            self._record(
                status="failed",
                run_id=template_snapshot.run_id,
                error_code=ErrorCode.VALIDATION_ERROR.value,
                reason="fixed_stage_sequence_mismatch",
            )
            raise GraphCompilerError(
                ErrorCode.VALIDATION_ERROR,
                "TemplateSnapshot.fixed_stage_sequence must match the Function One fixed stage order.",
            )

        definition = GraphDefinition(
            graph_definition_id=_graph_definition_id(template_snapshot.run_id),
            run_id=template_snapshot.run_id,
            template_snapshot_ref=template_snapshot.snapshot_ref,
            runtime_limit_snapshot_ref=runtime_limit_snapshot.snapshot_id,
            runtime_limit_source_config_version=runtime_limit_snapshot.source_config_version,
            stage_nodes=tuple(
                self._build_stage_nodes(template_snapshot.auto_regression_enabled)
            ),
            stage_contracts=self._build_stage_contracts(runtime_limit_snapshot),
            interrupt_policy=self._build_interrupt_policy(),
            retry_policy=self._build_retry_policy(template_snapshot, runtime_limit_snapshot),
            delivery_routing_policy=self._build_delivery_routing_policy(),
            source_node_group_map=self._build_source_node_group_map(),
            created_at=self._now(),
        )
        self._record(
            status="succeeded",
            run_id=definition.run_id,
            graph_definition_id=definition.graph_definition_id,
        )
        return definition
```

```python
def _graph_definition_id(run_id: str) -> str:
    candidate = f"graph-definition-{run_id}"
    if len(candidate) <= 80:
        return candidate
    digest = sha256(candidate.encode("utf-8")).hexdigest()[:24]
    return f"graph-definition-{digest}"
```
