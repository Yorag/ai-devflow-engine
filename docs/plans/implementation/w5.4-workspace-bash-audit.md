# W5.4 Workspace Bash Tool And Audit Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` to implement this plan task-by-task. Use `superpowers:test-driven-development` for every production or test change. Fallback to `superpowers:executing-plans` only if subagents cannot be scheduled or the task boundary cannot be supplied precisely. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the first concrete `bash` workspace tool so runtime stages can run controlled command execution through the existing `ToolRegistry` / execution gate / risk gate stack, while enforcing project-derived allowlist rules, shell-free subprocess execution, structured output, diff capture, and tool-specific audit records.

**Architecture:** Keep W5.4 inside the AL04 tool lane by introducing a new `backend/app/workspace/bash.py` module that implements `BashTool`, `BashCommandAllowlist`, `BashExecutionResult`, and `run_bash_command()` without widening into registry or risk-owner files. The tool will execute tokenized commands with `subprocess.run(..., shell=False)` from the current run workspace only, derive allowed command prefixes from the repo README / `pyproject.toml` / `frontend/package.json`, reject non-allowlisted commands with `bash_command_not_allowed`, capture pre/post workspace file snapshots for `command_trace` and `file_edit_trace`, and use new thin `AuditService.record_tool_call()` / `record_tool_error()` wrappers to persist bash-specific outcome rows without replacing the execution-gate intent audit ref that already becomes the final `ToolResult.audit_ref`.

**Tech Stack:** Python 3.11+, `subprocess`, `shlex`, `hashlib`, `json`, `pathlib`, existing `ToolProtocol` / `ToolRegistry` / `ToolExecutionGate`, `WorkspaceManager`, `AuditService`, `TraceContext`, `RedactionPolicy`, pytest via `uv run`.

## Key Behavior Decisions

- `bash` remains the formal tool contract name, but implementation is a controlled subprocess adapter. It must never call `subprocess.run(..., shell=True)` and must never hand the raw command string to PowerShell / `cmd` / Unix shell for free-form interpretation.
- The tool input contract is intentionally narrow:
  - `command`: required command string
- The tool result contract is intentionally bounded:
  - `command`
  - `argv`
  - `exit_code`
  - `duration_ms`
  - `stdout_excerpt`
  - `stderr_excerpt`
  - `stdout_truncated`
  - `stderr_truncated`
  - `changed_files`
- `BashCommandAllowlist` derives allowlisted command prefixes from workspace-local sources only:
  - README fenced command lines from `README.md` / `README.zh.md`
  - backend test commands implied by `pyproject.toml`
  - frontend script commands implied by `frontend/package.json`
  - a small fixed set of environment probes needed by the repo, such as `python --version`, `uv --version`, `node --version`, `npm --version`, `git --version`, and `rg --version`
- Commands not matched by the allowlist must fail with `ErrorCode.BASH_COMMAND_NOT_ALLOWED`.
- The existing W5.0d risk classifier remains owner of risk-level and `unknown_command` categorization. In practice:
  - known dependency-install / network / migration / env-change commands are still `high_risk` before reaching the tool and must require confirmation
  - commands that survive the execution gate but still fail the repo-derived allowlist are rejected by the concrete `bash` tool with `bash_command_not_allowed`
- Shell metacharacters that imply chained or redirected execution, such as `&&`, `||`, `|`, `>`, `>>`, `<`, `;`, and backticks, are treated as non-allowlisted command forms in this slice. This keeps the adapter deterministic and avoids reintroducing shell semantics behind the `bash` contract.
- Pre/post workspace snapshots are filesystem-based, not Git-dependent:
  - walk files under the current run workspace
  - exclude `.runtime/logs`, workspace-excluded paths, common dependency/build outputs, and directories outside the workspace
  - compute `sha256` per file
  - compare snapshots to produce changed relative paths
- Side-effect references follow existing AL04 conventions:
  - always add `command_trace:<run_id>:<call_id>`
  - add `file_edit_trace:<run_id>:<call_id>:<relative_path>` for each changed file
  - set `reconciliation_status = pending` when changed files exist, otherwise `not_required`
- The tool output must never echo secrets from stdout / stderr or environment-derived text. Use the same redaction policy class already used by workspace tools, with dedicated payload types for bash previews.
- `AuditService.record_tool_call()` and `record_tool_error()` are thin wrappers over the existing audit ledger and JSONL copy path. They do not create a second audit table or a second persistence protocol.
- Under the current W5.0b execution-gate implementation, the final `ToolResult.audit_ref` is normalized to the gate intent audit ref when `context.audit_recorder` is present. W5.4 therefore supplements the audit ledger with bash outcome rows, but it does not try to replace the final `ToolResult.audit_ref`.

## Log & Audit Integration

W5.4 changes tool execution, workspace mutation tracking, and security-sensitive command auditing, so this slice must stay inside the existing log/audit stack:

- Runtime log recording remains in `ToolExecutionGate.record_tool_result()`; `BashTool` must not create a parallel run-log writer.
- `BashTool` must receive the execution-gate `side_effect_intent_ref` and include it in bash outcome audit metadata. The final `ToolResult.audit_ref` remains the execution-gate intent audit ref under the current tool stack.
- `AuditService.record_tool_call()` must persist:
  - `action`: tool-scoped action such as `tool.bash.succeeded`
  - `target_type`: `tool_action`
  - `target_id`: stable command-trace target
  - structured metadata including command preview, exit code, duration, changed files, and truncated output excerpts
- `AuditService.record_tool_error()` must persist blocked or failed bash outcomes without leaking raw secrets or full stderr.
- Sensitive fields to redact or block:
  - Authorization headers
  - cookies
  - API keys
  - tokens
  - passwords
  - private-key material
- Audit failure behavior:
  - if a required post-execution audit write fails, the tool must return `tool_audit_required_failed`
  - do not invent partial success semantics that claim the audit exists when it does not
- Logs and audit rows remain observability artifacts only. They do not replace `StageArtifact`, domain events, Inspector truth, or future `ChangeSet` truth sources.

def test_bash_executes_allowlisted_build_command_shell_free_and_tracks_changed_files(tmp_path: Path) -> None:
    manager, workspace = _build_workspace(tmp_path)
    audit = _RecordingAudit()
    runner = _RecordingRunner(workspace.root)
    registry = ToolRegistry(
        [BashTool(manager=manager, workspace=workspace, audit_service=audit, runner=runner)]
    )

    result = registry.execute(
        _request("npm --prefix frontend run build"),
        _context(manager, workspace),
    )

    assert result.status is ToolResultStatus.SUCCEEDED
    assert result.output_payload["exit_code"] == 0
    assert result.output_payload["changed_files"] == ["frontend/dist/app.js"]
    assert result.side_effect_refs == [
        "command_trace:run-1:call-bash",
        "file_edit_trace:run-1:call-bash:frontend/dist/app.js",
    ]
    assert result.audit_ref is not None
    assert result.audit_ref.action == "tool.intent"
    assert audit.calls[0]["tool_name"] == "bash"
    assert runner.calls[0]["cwd"] == workspace.root
    assert runner.calls[0]["argv"] == ["npm", "--prefix", "frontend", "run", "build"]

@dataclass(frozen=True, slots=True)
class BashCommandAllowlist:
    allowed_prefixes: tuple[tuple[str, ...], ...]

    @classmethod
    def from_workspace(cls, workspace_root: Path) -> "BashCommandAllowlist":
        prefixes = {
            ("uv", "run", "pytest"),
            ("uv", "run", "python", "-m", "pytest"),
            ("npm", "--prefix", "frontend", "run", "build"),
            ("npm", "--prefix", "frontend", "run", "test"),
            ("npm", "--prefix", "frontend", "run", "dev"),
            ("python", "--version"),
            ("uv", "--version"),
            ("node", "--version"),
            ("npm", "--version"),
            ("git", "--version"),
            ("rg", "--version"),
        }
        return cls(allowed_prefixes=tuple(sorted(prefixes)))

    def is_allowed(self, argv: Sequence[str]) -> bool:
        if not argv:
            return False
        tupled = tuple(argv)
        return any(
            len(tupled) >= len(prefix) and tupled[: len(prefix)] == prefix
            for prefix in self.allowed_prefixes
        )

def run_bash_command(
    manager: WorkspaceManager,
    workspace: RunWorkspace,
    command: str,
    *,
    audit_service: BashAuditPort | None,
    runner: BashRunner | None,
    tool_input: ToolInput,
) -> ToolResult:
    argv = _parse_command(command)
    allowlist = BashCommandAllowlist.from_workspace(workspace.root)
    if not allowlist.is_allowed(argv):
        return _bash_not_allowed_result(tool_input, command=command, argv=argv)

    before = _workspace_snapshot(workspace)
    started = time.monotonic()
    execution = _default_runner if runner is None else runner
    completed = execution(argv, cwd=workspace.root, timeout=tool_input.timeout_seconds)
    duration_ms = max(0, int((time.monotonic() - started) * 1000))
    after = _workspace_snapshot(workspace)
    changed_files = _changed_files(before, after)

    stdout_excerpt, stdout_truncated = _truncate_output(str(completed["stdout"]), 8000)
    stderr_excerpt, stderr_truncated = _truncate_output(str(completed["stderr"]), 8000)
    bash_result = BashExecutionResult(
        argv=tuple(argv),
        returncode=int(completed["returncode"]),
        stdout=stdout_excerpt,
        stderr=stderr_excerpt,
        duration_ms=duration_ms,
        changed_files=tuple(changed_files),
    )
    audit_failure = _record_tool_call(
        audit_service,
        tool_input=tool_input,
        command=command,
        execution=bash_result,
    )
    if isinstance(audit_failure, ToolResult):
        return audit_failure
    return ToolResult(
        tool_name="bash",
        call_id=tool_input.call_id,
        status=ToolResultStatus.SUCCEEDED,
        output_payload={
            "command": command,
            "argv": list(bash_result.argv),
            "exit_code": bash_result.returncode,
            "duration_ms": bash_result.duration_ms,
            "stdout_excerpt": stdout_excerpt,
            "stderr_excerpt": stderr_excerpt,
            "stdout_truncated": stdout_truncated,
            "stderr_truncated": stderr_truncated,
            "changed_files": list(changed_files),
        },
        output_preview=_preview_bash(command, stdout_excerpt, stderr_excerpt),
        side_effect_refs=_side_effect_refs(workspace, tool_input, changed_files),
        reconciliation_status=(
            ToolReconciliationStatus.PENDING
            if changed_files
            else ToolReconciliationStatus.NOT_REQUIRED
        ),
        trace_context=tool_input.trace_context,
        coordination_key=tool_input.coordination_key,
    )

def _parse_command(command: str) -> list[str]:
    argv = shlex.split(command, posix=True)
    if not argv or any(token in _SHELL_META_TOKENS for token in argv):
        return []
    return argv


def _truncate_output(text: str, limit: int) -> tuple[str, bool]:
    redacted = RedactionPolicy(
        max_text_length=max(limit, 32),
        excerpt_length=max(limit, 32),
    ).summarize_text(
        text,
        payload_type="bash_output_preview",
    )
    if redacted.redaction_status.value == "blocked":
        return "[redacted]", False
    visible = str(redacted.redacted_payload or "")
    if len(visible) <= limit:
        return visible, False
    return f"{visible[: limit - 3]}...", True


def _workspace_snapshot(workspace: RunWorkspace) -> dict[str, str]:
    snapshot: dict[str, str] = {}
    for path in workspace.root.rglob("*"):
        if not path.is_file():
            continue
        relative = path.relative_to(workspace.root).as_posix()
        if any(
            relative == prefix or relative.startswith(f"{prefix}/")
            for prefix in (*workspace.excluded_relative_paths, *_EXCLUDED_PATH_PREFIXES)
        ):
            continue
        snapshot[relative] = hashlib.sha256(path.read_bytes()).hexdigest()
    return snapshot


def _changed_files(before: dict[str, str], after: dict[str, str]) -> list[str]:
    paths = set(before) | set(after)
    return sorted(path for path in paths if before.get(path) != after.get(path))
