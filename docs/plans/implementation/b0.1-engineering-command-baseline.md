# B0.1 Engineering Command Baseline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Do not create worktrees, switch branches, commit, merge, tag, push, or open a PR/MR from this plan.

**Goal:** Establish the backend and frontend dependency manifests, test commands, and development script entry points for B0.1 without implementing backend API behavior or frontend page behavior.

**Architecture:** B0.1 creates the minimum engineering surface that later slices consume. The backend remains a Python package with pytest configuration and no FastAPI app entry; B0.2 owns `backend/app/main.py`. The frontend receives a Vite/Vitest TypeScript baseline and a non-visual `main.tsx` bootstrap marker; F0.1 owns the React SPA routes, QueryClient, pages, and visible UI.

**Tech Stack:** Python 3.11+, pytest, pytest-asyncio, FastAPI dependency baseline, Pydantic v2, SQLAlchemy 2.x, Alembic, LangGraph 1.x, LangChain 1.x, httpx, React, Vite, TypeScript, Vitest, jsdom, React Router, TanStack Query, Zustand, npm.

- LangChain v1 source trace: official LangChain v1 migration guidance changes the prebuilt agent import from `langgraph.prebuilt.create_react_agent` to `langchain.agents.create_agent`, renames the static prompt parameter to `system_prompt`, moves hook-style customizations into middleware, changes structured output to `ToolStrategy` / `ProviderStrategy`, and moves runtime dependency injection to a `context` argument. Later runtime slices must use the v1 API surface and must not copy 0.3-era LangChain implementation details.
- Frontend dependency source: `docs/specs/frontend-workspace-global-design-v1.md` section `1.2`, which fixes React, Vite, React Router, TanStack Query, Zustand, and an EventSource wrapper as the frontend direction. This slice installs the engineering dependencies only; F0.1 implements SPA routing and UI.
- Boundary: this slice does not create `backend/app/main.py`, `backend/app/api/*` route files, React pages, app router, QueryClient, global CSS, API client, mocks, or business UI.
- Log & Audit Integration: not applicable. B0.1 only adds project manifests, test config, and non-runtime bootstrap files. It does not add user command interfaces, run lifecycle behavior, runtime nodes, model/tool calls, workspace writes, bash execution logic, Git delivery logic, remote delivery logic, product configuration behavior, or safety-sensitive runtime failures.
- API/OpenAPI checklist: not applicable. B0.1 does not modify `backend/app/api/routes/*` and does not expose FastAPI routes.
- Frontend Design Gate: not applicable. B0.1 does not implement visible frontend UI/UX. F0.1 is the first visible frontend slice and owns the project tone record.
