## Behavior Details

Request schema `PipelineTemplateWriteRequest`:
- `name: str` min length 1.
- `description: str | None`.
- `fixed_stage_sequence: list[StageType]`.
- `stage_role_bindings: list[StageRoleBinding]`.
- `approval_checkpoints: list[ApprovalType]`.
- `auto_regression_enabled: bool`.
- `max_auto_regression_retries: int` ge 0.

`name` and `description` are `PipelineTemplate` resource metadata required for user template creation and listing. They are not runtime slot configuration. C2.4 runtime-editable template fields remain limited to `role_id`, `system_prompt`, `provider_id`, `auto_regression_enabled`, and `max_auto_regression_retries`.

Validation rules:
- `fixed_stage_sequence` must exactly equal `FIXED_STAGE_SEQUENCE`.
- `approval_checkpoints` must exactly equal `FIXED_APPROVAL_CHECKPOINTS`.
- `stage_role_bindings` must contain exactly one binding per fixed stage, in fixed stage order.
- Each binding `role_id` must reference an existing role applicability entry in `ROLE_STAGE_TYPES`, and `binding.stage_type` must be present in `ROLE_STAGE_TYPES[binding.role_id]`. `STAGE_ROLE_IDS` remains the default seed binding source, not the full validation boundary.
- Each binding `provider_id` must reference an existing `ProviderModel`. The service may call `ProviderService.seed_builtin_providers()` before validation so the two builtin providers exist.
- Each binding `system_prompt` must be non-empty after stripping; the saved value is stripped.
- `max_auto_regression_retries` is stored as provided. C2.8 owns platform hard-limit validation.
- System prompt validation must pass through `TemplateService.validate_template_prompts_before_save()` exactly once per write command before persistence.
- Write commands must not accept `template_source`, `base_template_id`, `role_name`, stage order changes, approval checkpoint changes, prompt asset ids, prompt versions, runtime instructions, compression prompts, or provider record mutations.

Service behavior:
- `save_as_user_template(source_template_id: str | None, body: PipelineTemplateWriteRequest, trace_context: TraceContext) -> PipelineTemplateModel`
  - For `POST /api/pipeline-templates`, pass `source_template_id=None`; the created row has `base_template_id=None`.
  - For `POST /api/pipeline-templates/{templateId}/save-as`, require the source template exists; the created row has `base_template_id=templateId`.
  - Always creates a new `template_id` with `template-user-<uuid hex>`.
  - Always sets `template_source=TemplateSource.USER_TEMPLATE`.
  - Records `template.save_as` success audit with metadata containing `template_id`, `source_template_id`, `base_template_id`, `template_source`, `stage_types`, `role_ids`, `provider_ids`, `auto_regression_enabled`, and `max_auto_regression_retries`; metadata must not include prompt text or prompt asset metadata.
- `patch_user_template(template_id: str, body: PipelineTemplateWriteRequest, trace_context: TraceContext) -> PipelineTemplateModel`
  - Missing template returns `not_found`, 404.
  - `system_template` returns `validation_error`, 409, with rejected audit action `template.patch.rejected`.
  - `user_template` updates allowed fields, keeps the same `template_id`, `template_source`, `base_template_id`, and `created_at`, updates `updated_at`, and records `template.patch`.
- `delete_user_template(template_id: str, trace_context: TraceContext) -> None`
  - Missing template returns `not_found`, 404.
  - `system_template` returns `validation_error`, 409, with rejected audit action `template.delete.rejected`.
  - `user_template` delete first rejects the command with `validation_error`, 409 if any other `PipelineTemplateModel.base_template_id` references the template, because delete must preserve template lineage and avoid dangling self-FK references. If no child template references it, delete moves every `SessionModel` with `selected_template_id = template_id`, `status = draft`, and `current_run_id = null` to `DEFAULT_TEMPLATE_ID`, then removes the user template row. If any non-draft or already-started Session still references the user template, delete is rejected with `validation_error`, 409, because C2.4 must not alter started run semantics or future snapshot references.
  - The delete path records `template.delete` before commit and rolls back the fallback updates plus delete if audit fails. The success audit metadata includes `fallback_template_id` and `fallback_session_ids` but no prompt text.

Error messages:
- Missing template: `Pipeline template was not found.`
- Patch system template: `System templates cannot be overwritten.`
- Delete system template: `System templates cannot be deleted.`
- Invalid template definition: `Pipeline template contains invalid editable fields.`
- Missing provider: `Pipeline template references an unknown Provider.`
- Delete referenced by started Session: `Pipeline template is selected by a Session that has already started.`
- Delete referenced by child template: `Pipeline template is used as a base template by another template.`

Log & Audit Integration:
- Runtime log category: not applicable; C2.4 writes control-plane configuration and does not create run logs.
- Audit actions:
  - Success: `template.save_as`, `template.patch`, `template.delete`.
  - Rejected: `template.save_as.rejected`, `template.patch.rejected`, `template.delete.rejected`.
- Actor: `AuditActorType.USER`, `actor_id="api-user"` to match C2.1-C2.3 control-plane command patterns.
- Target: `target_type="pipeline_template"`, `target_id` is the new/existing/requested template id.
- Trace fields: all route calls pass `get_trace_context()` into service methods. Audit records inherit `request_id`, `trace_id`, `correlation_id`, `span_id`, and `parent_span_id` through `AuditService`.
- Redaction: audit metadata includes ids, enum values, counts, and booleans only. It must not include full `system_prompt`, prompt asset ids/versions, credentials, API keys, local absolute paths, or provider secret values.
- Audit failure behavior: because template saves, patches, deletes, and invalid system-template write attempts are configuration/security-sensitive control-plane commands, audit ledger failure must raise and rollback the control write or rejected-command response. Tests must prove a success audit failure leaves no created/updated/deleted user template row.
- Product truth: audit records are only security/control-plane facts; `PipelineTemplateModel` remains the template truth. Tests must not read audit logs as a substitute for template list/detail behavior.

```python
class PipelineTemplateWriteRequest(_StrictBaseModel):
    name: str = Field(min_length=1)
    description: str | None = None
    fixed_stage_sequence: list[common.StageType]
    stage_role_bindings: list[StageRoleBinding] = Field(min_length=1)
    approval_checkpoints: list[common.ApprovalType]
    auto_regression_enabled: bool
    max_auto_regression_retries: int = Field(ge=0)

    @field_validator("fixed_stage_sequence")
    @classmethod
    def require_fixed_stage_sequence(
        cls,
        value: list[common.StageType],
    ) -> list[common.StageType]:
        if tuple(value) != FIXED_STAGE_SEQUENCE:
            raise ValueError(
                "fixed_stage_sequence must match the Function One V1 stage order"
            )
        return value

    @field_validator("approval_checkpoints")
    @classmethod
    def require_fixed_approval_checkpoints(
        cls,
        value: list[common.ApprovalType],
    ) -> list[common.ApprovalType]:
        if tuple(value) != FIXED_APPROVAL_CHECKPOINTS:
            raise ValueError(
                "approval_checkpoints must contain the two Function One V1 checkpoints"
            )
        return value
```

```python
        self.records.append({"method": "record_command_result", **kwargs})
```

```python
    def validate_template_prompts_before_save(
        self,
        bindings: list[dict[str, str]],
    ) -> list[dict[str, str]]:
        validated: list[dict[str, str]] = []
        for binding in bindings:
            prompt = binding["system_prompt"].strip()
            if not prompt:
                raise TemplateServiceError(
                    ErrorCode.VALIDATION_ERROR,
                    INVALID_TEMPLATE_EDITABLE_FIELDS_MESSAGE,
                    422,
                )
            validated.append({**binding, "system_prompt": prompt})
        return validated

    def validate_editable_fields(
        self,
        body: PipelineTemplateWriteRequest,
        *,
        trace_context: TraceContext,
    ) -> list[dict[str, str]]:
        ProviderService(
            self._session,
            audit_service=self._audit_service,
            now=self._now,
        ).seed_builtin_providers(trace_context=trace_context)
        provider_ids = {
            provider_id
            for (provider_id,) in self._session.query(ProviderModel.provider_id).all()
        }
        expected_stages = [stage.value for stage in FIXED_STAGE_SEQUENCE]
        if [binding.stage_type.value for binding in body.stage_role_bindings] != expected_stages:
            raise TemplateServiceError(
                ErrorCode.VALIDATION_ERROR,
                INVALID_TEMPLATE_EDITABLE_FIELDS_MESSAGE,
                422,
            )

        normalized: list[dict[str, str]] = []
        for binding in body.stage_role_bindings:
            applicable_stage_types = ROLE_STAGE_TYPES.get(binding.role_id, [])
            if binding.stage_type not in applicable_stage_types:
                raise TemplateServiceError(
                    ErrorCode.VALIDATION_ERROR,
                    INVALID_TEMPLATE_EDITABLE_FIELDS_MESSAGE,
                    422,
                )
            if binding.provider_id not in provider_ids:
                raise TemplateServiceError(
                    ErrorCode.VALIDATION_ERROR,
                    UNKNOWN_TEMPLATE_PROVIDER_MESSAGE,
                    422,
                )
            normalized.append(
                {
                    "stage_type": binding.stage_type.value,
                    "role_id": binding.role_id,
                    "system_prompt": binding.system_prompt,
                    "provider_id": binding.provider_id,
                }
            )
        return self.validate_template_prompts_before_save(normalized)
```
