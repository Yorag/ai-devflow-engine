## Implementation Summary

- Added `WorkspaceProjectionService` in `backend/app/services/projections/workspace.py`.
- Added `GET /api/sessions/{sessionId}/workspace` in `backend/app/api/routes/query.py` and registered it from `backend/app/api/router.py`.
- The projection returns the existing strict `SessionWorkspaceProjection` schema.
- The service reads visible `Session`, `Project`, and delivery channel facts from `control.db`, `PipelineRun` and current `StageRun` facts from `runtime.db`, and top-level Narrative Feed entries from `EventStore.list_for_session()` in `event.db`.
- Removed sessions and sessions under removed projects return stable 404 semantics with `Session workspace was not found.`.
- `waiting_tool_confirmation` projects a top-level `tool_confirmation` feed entry and disables normal Composer input with `primary_action = pause`.
- Run snapshot semantics are preserved by using `runs[].run_id` as the stable locator for the `PipelineRun` row and its persisted snapshot refs. The service does not read latest `PlatformRuntimeSettingsModel` to reinterpret historical runs.
