- `ToolConfirmationService` now owns request creation, top-level `tool_confirmation` projection building, allow/deny command validation, runtime resume coordination, and terminal-run cancellation.
- Tool-confirmation projections keep command actions scoped to the specific `tool_confirmation_id` with `allow:{id}` and `deny:{id}` semantics.
- The tool-confirmation API exposes dedicated allow/deny routes with distinct request schemas, response schemas, and OpenAPI components.
- Route dependency wiring prefers `app.state.h44a_runtime_port` and `app.state.h44a_audit_service` while preserving existing fallbacks.

- Service regressions now cover duplicate visible sessions sharing `current_run_id`, required-audit failure rollback, runtime-resume failure rollback, and cancellation commit rollback.
