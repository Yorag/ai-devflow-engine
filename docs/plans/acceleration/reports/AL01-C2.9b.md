## Implemented Changes

1. Service-layer remove command

- Added Project history remove command helpers in `backend/app/services/projects.py`.
- Successful remove soft-hides the Project plus still-visible Sessions without mutating runtime/event/log facts.
- Default Project rejection remains a `409` validation error.
- Missing Project, repeated removal, active-run blocked removal, and default-Project rejection all use stable error/result semantics with audit records.

2. Runtime blocking and audit hardening

- Reused `RunStateMachine.is_active_run_status()` plus pending startup publication semantics for active-run truth.
- Added project-wide runtime blocker scan to catch stale or not-yet-refreshed `current_run_id` tails.
- Added runtime-side write barrier around remove so concurrent runtime writes cannot slip in after blocker scan and before visibility commit.
- Hardened success-audit compensation:
  - pre-ledger audit failure restores Project/Session visibility and leaves no success audit row
  - after-ledger audit-copy failure is treated as success because the authoritative success audit row exists

3. API route and OpenAPI

- Added `DELETE /api/projects/{projectId}` in `backend/app/api/routes/projects.py`.
- Route returns `ProjectRemoveResult` on `200` for both success and active-run blocked removal.
- Route maps `404`, `409`, `422`, and `500` to `ErrorResponse`.
- OpenAPI coverage verifies route presence, no request body, response set, `ProjectRemoveResult`, and destructive-operation safety flags.
