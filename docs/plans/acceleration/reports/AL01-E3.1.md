## Implementation Summary

- Added service-layer `DomainEvent`, `DomainEventType`, `EventProjectionTarget`, `EventProjectionRule`, `EventProjectionMatrix`, and resolver helpers in `backend/app/services/events.py`.
- Added `EventStore.append()`, `EventStore.list_after()`, and `EventStore.list_for_session()`.
- `EventStore.append()` validates payloads through `SessionEvent` before persistence, writes canonical JSON payloads, inherits or verifies `TraceContext` identifiers, stores `correlation_id`, flushes without committing, and preserves caller-owned transaction control.
- Required E3.1 terminal, tool confirmation, provider retry/circuit breaker, delivery, approval, and run status mappings are covered.
- `ProjectLoaded` is recognized as a formal domain event name but has no session SSE projection in E3.1 and is not mapped to `session_created`.
- Raw graph events such as `GraphNodeStarted` are rejected as external domain events.
- Sequence allocation uses a process-local high-water guard per event database URL and session id to avoid same-process duplicate sequence indexes without changing the C1.9 database model.
