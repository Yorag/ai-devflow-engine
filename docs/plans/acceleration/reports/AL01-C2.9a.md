## Implemented Changes

1. Service-layer delete command

- Added Session history delete command helpers in `backend/app/services/sessions.py`.
- Successful delete now soft-hides the Session without mutating runtime/event/log facts.
- Blocked delete returns structured `SessionDeleteResult` on active runs and pending startup publications.
- Repeated delete and missing Session produce stable rejected error semantics with audit records.

2. Concurrency and audit hardening

- Added delete-side guard against pending startup publication races.
- Hardened `PublicationBoundaryService.begin_startup_publication()` with guarded `INSERT ... SELECT` so a hidden Session cannot acquire a new pending startup publication after delete.
- Fixed same-session identity-map visibility after delete.
- Refined audit failure semantics:
  - primary audit ledger failure restores visibility and leaves no success audit row
  - after-ledger audit-copy metadata failure is treated as success because the authoritative success audit row exists

3. API route and OpenAPI

- Added `DELETE /api/sessions/{sessionId}` in `backend/app/api/routes/sessions.py`.
- Route returns `SessionDeleteResult` on `200` for both success and blocked delete.
- Route maps `404`, `409`, `422`, and `500` to `ErrorResponse`.
- OpenAPI coverage verifies the route schema and absence of request body.
