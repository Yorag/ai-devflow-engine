## Scope

A4.7 adds a standalone LangGraph event translation boundary. It converts explicit node started/completed and interrupt facts into product-facing domain events, `StageArtifact.process` records, stable process refs, and sanitized runtime log summaries.

This worker did not wire the translator into `langgraph_engine.py`, did not change API/projection/frontend/database files, and did not update central acceleration status files or the shared coordination store.

## Implementation Evidence

- Added `LangGraphEventTranslator` with typed fact models for node started, node completed, and interrupts.
- Node started facts emit `StageStarted`, append `langgraph_node_started` process records, and write sanitized runtime logs.
- Node completed facts emit `StageUpdated`, append `langgraph_node_completed` process records, and write sanitized runtime logs.
- Clarification and approval interrupts emit the matching domain events and use `langgraph_interrupt:<interrupt_id>` process keys.
- Tool-confirmation interrupts emit `ToolConfirmationRequested` only when supplied a valid product-facing feed payload.
- Missing tool-confirmation payload writes process/log evidence without a domain event.
- Invalid or identity-mismatched tool-confirmation payload writes process/log evidence with `domain_event_ref=None`, does not append a domain event, then raises `LangGraphEventTranslationError`.
- Explicit tool-confirmation payload identity is bound to `run_id`, `stage_run_id`, and `tool_confirmation_id` before event append.
- Raw graph event names, raw graph state, checkpoint payloads, compiled graph objects, raw event refs, and raw payload excerpts are not written to product payloads, process records, or log summaries.
