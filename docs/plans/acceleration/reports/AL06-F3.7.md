## Review

Spec / plan compliance review result:

- Approved with no Critical or Important findings.
- Verified grouped render order, `approval_result` remaining non-openable, stage trace/reference visibility, dedicated tool-confirmation projection usage, hidden inapplicable metrics, and `request` passthrough into the rail.

Initial code quality / testing / regression review result:

- Two Important findings were raised and verified.
- Finding 1: Inspector detail fetched once per target and could become stale while the same run kept advancing through SSE/store updates.
- Finding 2: document-scope `Escape` close could also clear the Inspector while focus sat inside the modal settings dialog.

Code quality fix result:

- `InspectorPanel` now derives a refresh key from the live same-run feed entries for stage targets and from the matching top-level entry for control-item, tool-confirmation, and delivery-result targets, so relevant SSE increments trigger a refetch of the open detail.
- `InspectorPanel` now ignores `Escape` events originating inside a modal dialog (`role="dialog"` with `aria-modal="true"`), preserving settings close behavior without clearing the inspector behind it.
- New regressions cover stage refresh after `stage_updated`, stage refresh after same-run `approval_result`, and modal `Escape` while the inspector remains open.

Final code quality re-review result:

- Approved with no remaining Critical or Important findings.
