## Review

Spec / plan compliance review result:

- No Critical or Important findings.
- The review confirmed this slice stayed inside `F4.1`, preserved `draft` / `waiting_clarification` send semantics, mapped runtime states to pause/resume/disabled presentation correctly, and did not introduce F4.2 control actions.

Initial code quality / testing / regression review result:

- Two Important findings were raised.
- Finding 1: sendable states were derived from `mode` alone and did not fail closed when `is_input_enabled` or `primary_action` contradicted the projection.
- Finding 2: terminal / disabled states did not take precedence over contradictory `primary_action: "resume"`.

Code quality fix result:

- `resolveComposerState()` now only treats `draft` / `waiting_clarification` as sendable when `primary_action === "send"` and `is_input_enabled === true`.
- `resolveComposerState()` now resolves readonly / disabled states before contradictory resume semantics.
- Focused regression tests were added for both contradictory projection classes.

Code quality re-review result:

- No Critical or Important findings in re-review.
