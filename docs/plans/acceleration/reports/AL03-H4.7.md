## Implementation Summary

- `RunLifecycleService` now owns rerun current-tail loading, run-state validation, runtime old-thread terminal confirmation, new run creation, session pointer movement, `PIPELINE_RUN_CREATED` emission, session-status change emission, rerun audit/log integration, and append-only terminal `retry_action` refresh.
- Rerun now tolerates a terminal current tail with `current_stage_run_id=None`, using the run boundary itself for runtime terminal validation and for audit/log metadata when no stage row exists.
- Workspace projection fallback now mirrors timeline behavior: only an active non-terminal run without a stage row may borrow `session.latest_stage_type`; a terminal current tail no longer leaks an active-stage label.
- The rerun API exposes only `POST /api/sessions/{sessionId}/runs`, preserves body-less POST behavior, and documents the strict `SessionRerunRequest` plus `RunCommandResponse` OpenAPI shape.
- Because runtime/control/event stores commit independently in this repository, H4.7 now includes a bounded compensation path for rerun when control/event commit fails after runtime commit: restore the session pointer first if needed, then delete the new runtime row, and surface a failed command if compensation cannot complete.
