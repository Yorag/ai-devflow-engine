## Implementation Summary

- Added `StageAgentRuntime` as a `StageNodeRunnerPort` implementation that initializes stage input, builds a fresh `ContextEnvelope` each iteration, invokes `LangChainProviderAdapter.invoke_with_retry()`, parses decisions through `AgentDecisionParser`, and writes stage output through `ArtifactStore.complete_stage_output()`.
- Routed tool decisions through `ToolRegistry.execute()` with `ToolExecutionContext`, current stage contracts, timeout, workspace boundary, audit recorder, run-log recorder, risk policy, and confirmation port.
- Persisted stage process records for start, model call, provider retry, provider circuit breaker, decision, tool, tool confirmation, structured repair, recovery checkpoint, side-effect reconciliation, and failure paths.
- Added structured repair routing for repair decisions and repairable parser errors, bounded by `max_structured_output_repair_attempts`.
- Added `retry_with_revised_plan` continuation semantics within the normal ReAct iteration limit.
- Filtered `model_call_trace` so prompt/model summary excerpts and redacted payload bodies are not copied into process records.
- Passed stage artifacts, context references, change sets, clarifications, and approval decisions into `ContextBuildRequest`.
- Updated LangGraph stage node handling so non-completed results are not marked completed and a later `run_next()` cannot advance to a successor business node after a waiting/failed result.
