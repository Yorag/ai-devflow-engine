## Implementation Summary

- Added `PromptValidationRule`, `PromptValidationResult`, `PromptValidationError`, and `PromptValidationService` under `backend/app/runtime/prompt_validation.py`.
- `PromptValidationService` derives stage contracts from the same `GraphCompiler` source used by `GraphDefinition.stage_contracts`; it does not create a parallel stage-rule table.
- Template-save validation and run-start validation now use the same prompt-validation boundary.
- Rejection rules now cover:
  - platform instruction override
  - unauthorized tool override
  - approval bypass
  - clarification bypass
  - structured-output disablement
  - output-schema override
  - credential leakage
  - raw chain-of-thought leakage
  - delivery override
  - audit bypass
  - stage-contract override
  - stage tool-boundary violations
  - blank prompts
  - over-length prompts
  - runtime context-budget overflow
- The tool-boundary matcher now accepts safe negated guidance such as “Do not use bash” while still rejecting later positive disallowed-tool instructions in the same prompt.
- `TemplateService` now reads either the existing runtime settings row or an unsaved default read model for prompt validation, avoiding runtime-settings initialization side effects on rejected template saves.
- `SessionService`’s default run-start adapter now validates frozen prompt snapshots through `PromptValidationService` and preserves validation-class error mapping to `422`.
