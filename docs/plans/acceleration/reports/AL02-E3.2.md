## Implementation Summary

- Extracted the session SSE route from `backend/app/api/routes/query.py` into the dedicated AL02-owned module `backend/app/api/routes/events.py`.
- `query.py` now includes the extracted router and no longer contains inline SSE polling, frame serialization, or replay cursor logic.
- Added `SseEventEncoder` to `backend/app/services/events.py` so SSE frame formatting is centralized and derived directly from `DomainEvent`.
- The encoder emits the existing session event envelope fields `event_id`, `session_id`, `run_id`, `event_type`, `occurred_at`, and `payload`, and includes top-level `correlation_id` only when present for diagnostics.
- Replay remains backed by `EventStore.list_after()` with combined `Last-Event-ID` and `after` cursor handling, bounded `limit`, and keep-alive comments.
- Added `backend/tests/api/test_sse_stream.py` covering encoder behavior, replay ordering after reconnect, provider retry/circuit-breaker stage semantics, `tool_confirmation_requested` and `tool_confirmation_result` payload semantics, and OpenAPI route documentation.
