## Implemented Changes

1. Control-plane publication gate

- Added `StartupPublicationModel` in `backend/app/db/models/control.py`.
- Added `backend/app/domain/publication_boundary.py` constants and `PublishedStartupVisibility`.
- Added `PublicationBoundaryService` in `backend/app/services/publication_boundary.py`.
- Contract:
  - `pending_session_id` is the per-session startup claim guard
  - `publish_startup_visibility()` flips Session public state and publication state together
  - `abort_startup_publication()` deletes staged runtime/graph/event truth and marks the publication `aborted`
  - retry is explicit `409 until cleanup`; no undocumented stale pending hole remains

2. First-run startup integration

- Refactored `RunLifecycleService.start_first_run()` to:
  - begin pending startup publication before staging truth
  - stage runtime, graph, and event rows
  - publish Session visibility only after staged-set verification succeeds
  - abort and cleanup staged truth on failures after begin
- Kept existing run-start business semantics intact for `R3.2`, including snapshots, graph/thread creation, first stage, first message event, and audit/log behavior.

3. Reader visibility enforcement

- `backend/app/services/projections/workspace.py`
  - hides runs/events whose run ids are in the startup hidden set
- `backend/app/services/projections/timeline.py`
  - rejects unpublished startup run ids with `404`
  - filters timeline replay against the same hidden set
- `backend/app/api/routes/events.py`
  - suppresses unpublished startup events in session SSE replay batches
