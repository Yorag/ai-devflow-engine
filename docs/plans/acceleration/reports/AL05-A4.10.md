## Implemented Changes

1. Auto regression runtime policy

- Added `AutoRegressionPolicy` with frozen retry-limit resolution across `TemplateSnapshot`, current run `RuntimeLimitSnapshot`, `GraphDefinition.retry_policy`, and `PlatformHardLimits`.
- Added top-level `resolve_max_auto_regression_retries()` and `should_retry_review_issue()` wrappers.
- Added strict graph retry policy validation for `return_stage_on_review_regression`, `max_auto_regression_retries`, and strict-bool `auto_regression_enabled`.

2. Code Review retry decision

- Schedules `changes_requested` Code Review artifacts with valid evidence back to `code_generation` through route key `review_regression_retry`.
- Allows approval only for stable decisions `approved` and `no_changes_requested`, or when frozen template auto regression is disabled.
- Blocks approval for missing evidence, unknown/malformed regression decisions, and retry exhaustion.
- Preserves A4.11 ownership of control records, retry stage runs, and exhausted-failure publication.

3. Sanitized logging and validation hardening

- Writes bounded runtime log summaries through an injected run log writer, inheriting the supplied `TraceContext`.
- Avoids logging raw review text, unknown decision text, workspace paths, credentials, prompts, model responses, tool output, or full issue payloads.
- Ignores log sink failures so logging cannot mask the policy decision.
- Rejects blank placeholders, arbitrary string placeholders, unstructured issue/fix entries, and unstable top-level evidence refs as missing review evidence.
- Enforces `AutoRegressionDecision` invariants for retry and non-retry decision shapes.
