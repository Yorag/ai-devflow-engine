## Implemented Changes

1. Retry control records

- Added `RetryControlResult`.
- Added `ControlRecordService.append_retry_control_item()`.
- Persists `RunControlRecordModel(control_type=retry)`.
- Emits `DomainEventType.RETRY_TRIGGERED` with payload shape `{"control_item": ...}`, preserving existing `control_item_created` event semantics.
- Persists `StageArtifactModel(artifact_type="control_item_trace")` so Inspector metrics expose `retry_index` and visible one-based `source_attempt_index`.
- Preserves first-retry policy context by writing `policy_source_attempt_index` into `process["output_snapshot"]`.
- Rejects invalid retry/source attempt indexes, including bool values.

2. Auto regression runner

- Added `AutoRegressionRunner`, `AutoRegressionRunnerResult`, `AutoRegressionExhaustedFailure`, `RetryControlRecorder`, and `CodeReviewApprovalCreator`.
- Converts A4.10 retry decisions into retry control items targeting `code_generation`.
- Returns explicit exhausted failure output with `StageStatus.FAILED` and does not create approval on retry exhaustion.
- Creates `code_review_approval` only for stable review decisions through the injected approval creator.
- Blocks `changes_requested` when auto regression is disabled instead of silently advancing to approval.
