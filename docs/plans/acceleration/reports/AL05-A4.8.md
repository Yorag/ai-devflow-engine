## Implementation Summary

- Added immutable `ProviderConfig` with frozen Provider snapshot id, model binding snapshot id, Provider identity, model id, model parameters, and all five model capability fields.
- Added sanitized `ProviderResolutionEvent` records for resolution success, failure, and missing credential reference without exposing `api_key_ref` contents.
- Added `ModelProvider` protocol for future Provider adapter integration.
- Added `ProviderRegistry` that resolves only from supplied `ProviderSnapshotRead` and `ModelBindingSnapshotRead` objects.
- Added `ProviderRegistry.resolve_from_template_snapshot()` selector for frozen run/template snapshot objects with `provider_snapshots` and `model_binding_snapshots` attributes or keys.
- Added structured registry errors for missing model binding snapshot, missing Provider snapshot, snapshot mismatch, and unsupported capability.
- Added tool-call capability guard through `requires_tool_calling=True`.
